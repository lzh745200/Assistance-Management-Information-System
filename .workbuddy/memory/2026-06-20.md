用户要求"修复所有BUG，确保系统完全没有问题"。通过运行 flake8/ESLint/mypy/vue-tsc/pytest 诊断，定位并修复了以下真实运行时BUG：

### 后端修复

1. **file_upload.py — magic 库 Windows 致命崩溃（阻塞全部后端测试）**
   - `import magic` 在 Windows 触发不可捕获的 access violation，导致整个 pytest 套件退出码 127
   - 修复：移除模块级 `import magic`，改用 stdlib `mimetypes` + 扩展名映射回退；`magic` 设为 None（测试 `mod.magic` 仍可用）
   - detect_mime_type/detect_mime_type_from_bytes 改为优先 mimetypes，magic 仅作可选增强

2. **feedback.py — 导入不存在的 auth.verify_token（用户提交反馈崩溃）**
   - 根因：`verify_token` 在 `auth/auth.py` 中定义但未从 `auth/__init__.py` 导出，导致生产环境永远走 ImportError→None 分支，用户反馈不记录身份
   - 修复：在 `app/api/v1/auth/__init__.py` 导出 `verify_token`（保留其黑名单+LRU缓存安全检查），feedback.py 维持延迟导入 + await
   - 关键：不能用 security.decode_token 替代（会丢失黑名单检查，且破坏 test_feedback_api.py 的 patch）

3. **reports.py — 6 处漏 await + 类型不匹配（报表导出全部崩溃）**
   - ReportService 的 export_to_excel/export_to_pdf/export_comprehensive_report/get_export_filename 均为 async，但路由层调用时漏 await（把协程传给 BytesIO）
   - ExportQuery/DrillDownQuery 是 Pydantic 模型，service 期望 dict → 用 model_dump() 转换
   - 修复点：export_excel/export_pdf/export_comprehensive_report(×2)/drill_down

4. **reports.py:186 — 元组解包 dict（帮扶村筛选崩溃）**
   - `villages, total = service.filter_villages(...)` 但 filter_villages 返回 dict（含 items/total/page 等键），会 "too many values to unpack"
   - 修复：`result = service.filter_villages(...); villages = result.get("items",[]); total = result.get("total",0)`

5. **admin.py:229-231 — int(None) 潜在崩溃**
   - `int(svc.get("max_login_attempts","5"))` 当配置值为 None 时 int(None) 抛 TypeError
   - 修复：`int(svc.get("max_login_attempts") or "5")` 防御性处理

6. **main.py:278-280 + reports.py — mypy 类型注解修复**
   - Optional[Path] 注解；filters dict 显式注解；SQLAlchemy Column 加 type:ignore

### 前端修复

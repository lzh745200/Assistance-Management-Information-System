## Part 1 — 本次交付摘要

### 阶段 0：样式体系卫生（零视觉变化）

- `styles/index.scss` 的 `--el-color-*` 硬编码改为 `var(--color-*)` 引用（仅无 token 对应的派生中间档保留原值并注释），消除"改色要改两处"的断层。
- 新增 `styles/tokens-vars.scss`（纯 SCSS 变量、无实体规则），`vite.config.ts` additionalData 改注入它——原先 621 行 tokens.scss（含 `:root`/`[data-theme]` 规则）随每个 SFC 重复打包的问题消除。
- 删除孤儿文件 `styles/military-theme.scss`（全仓无引用，tokens.scss 已有 `[data-theme="military"]`）。
- `AGENTS.md` 登记 tokens-vars.scss；`项目文件结构说明.md` 同步更新。

### 阶段 1：色彩统一

- dashboard：KpiCards 第三套色板收敛为 token 语义色；两处 Element 蓝描边改 `var(--el-color-primary)`；5 个假 sparkline（echarts 平线）删除；Unicode ↗↘→ 换 Top/Bottom/Minus 图标组件。
- schools/List："新增学校"按钮三重 `!important`+渐变覆盖删除，回归标准 primary；7 张彩虹统计卡统一为中性卡面 + 语义色数值。
- funds/EnhancedList：批量工具栏蓝色改军绿系，不再与军绿表头撞色。
- 全站蓝残留：命中 29 处，替换 6 处非语义蓝；**保留 23 处**状态/分类语义蓝（日志级别、数据冷热分层、图表系列色、地图 marker 等），待产品语义映射决策。

### 阶段 2：状态与可访问性（纯新增，成功路径零改动）

- error 态：dashboard 三组件 + schools/List + funds/EnhancedList 的静默 catch 全部补 `ElMessage.error` + 错误占位 + 重试按钮。
- loading：dashboard KPI/图表区加 el-skeleton；empty：el-table `#empty` 插槽统一 el-empty。
- a11y：可点统计卡补 `role="button" tabindex="0"` + 键盘触发；无行为卡片去除 cursor/hover 假可点。

### 阶段 3：响应式

- 侧边栏 <768px 自动收起（matchMedia，含卸载清理，不覆盖用户折叠偏好）。
- 各页 stats-row/header-actions 补 `flex-wrap`；KPI 卡 ≤480px 单列；操作列去 `fixed="right"` 保留宽度。

## Part 3 — 里程碑建议

- **第 1 周**：P0 全部（提交归位、测试噪音、三件套推广启动、目录合并）。
- **第 2–4 周**：P1 按 ①→③→②→⑤→④ 顺序（先清残留、再定语义、后接主题）。
- **迭代期**：P2 穿插排期；P2-④ 视觉回归建议在 P1-① 大批量改动**之前**落地，作为安全网。

### 执行纪律（沿用本次验证有效的方式）

1. 每批次四道门禁：eslint（--max-warnings=0）→ vue-tsc → 相关 vitest → build。
2. 等值替换先行，视觉变化项单列并人工核对页面。
3. 不新增依赖、不动三处高风险全局覆盖区（main.ts 运行时注入、DefaultLayoutSafe :deep 层、index.scss EP 覆盖）。
4. 每批次独立 commit，可整段回退。

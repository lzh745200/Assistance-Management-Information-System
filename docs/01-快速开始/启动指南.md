## 🚀 一键启动

### Windows 用户

#### 方法 1：使用批处理脚本（推荐）

双击运行 `一键启动.bat` 文件，脚本将自动完成以下操作：

1. ✅ 检查系统依赖（Python、Node.js、npm）
2. ✅ 安装后端依赖
3. ✅ 初始化数据库
4. ✅ 安装前端依赖
5. ✅ 启动后端服务（端口 8000）
6. ✅ 启动前端服务（端口 5173）

#### 方法 2：使用 Python 脚本

```bash
cd "d:\military-Rural Revitalization-system"
python scripts/start.py
```

启动模式选项：
- `all` - 启动前后端（默认）
- `backend` - 仅启动后端
- `frontend` - 仅启动前端

示例：
```bash
# 启动前后端
python scripts/start.py all

# 仅启动后端
python scripts/start.py backend

# 仅启动前端
python scripts/start.py frontend
```

### Linux/Mac 用户

```bash
cd /path/to/military-Rural-Revitalization-system
python scripts/start.py
```

## 🔍 代码质量检查

### Windows 用户

#### 方法 1：使用批处理脚本（推荐）

双击运行 `代码质量检查.bat` 文件

#### 方法 2：使用 Python 脚本

```bash
cd "d:\military-Rural Revitalization-system"
python scripts/testing/code_quality_analyzer.py
```

检查模式选项：
- 默认 - 仅检查
- `--fix` 或 `-f` - 自动修复部分问题
- `--test` 或 `-t` - 运行单元测试

示例：
```bash
# 仅检查
python scripts/testing/code_quality_analyzer.py

# 自动修复
python scripts/testing/code_quality_analyzer.py --fix

# 检查并运行测试
python scripts/testing/code_quality_analyzer.py --test

# 自动修复并运行测试
python scripts/testing/code_quality_analyzer.py --fix --test
```

### 检查项目

| 检查项 | 工具 | 说明 |
|---------|------|------|
| 代码格式 | Black | Python 代码格式化 |
| 导入排序 | isort | Python 导入语句排序 |
| 代码风格 | flake8 | PEP8 代码风格检查 |
| 类型检查 | mypy | Python 类型注解检查 |
| 单元测试 | pytest | 自动化测试 |

## ❓ 常见问题

### 1. 端口被占用

**问题**：`Address already in use`

**解决方案**：
- 修改后端端口：`uvicorn app.main:app --port 8001`
- 修改前端端口：`npm run dev -- --port 5174`

### 2. Python 模块未找到

**问题**：`ModuleNotFoundError: No module named 'xxx'`

**解决方案**：
```bash
cd backend
pip install -r requirements.txt
```

### 3. Node.js 依赖安装失败

**问题**：`npm install` 失败

**解决方案**：
```bash
cd frontend
rm -rf node_modules package-lock.json  # Linux/Mac
# 或
rmdir /s /q node_modules && del package-lock.json  # Windows

npm install
```

### 4. 数据库初始化失败

**问题**：`Failed to initialize database`

**解决方案**：
```bash
cd backend
# 删除现有数据库
del data\*.db  # Windows
# 或
rm data/*.db  # Linux/Mac

# 重新初始化
python init_db.py
```

### 5. CORS 错误

**问题**：浏览器控制台显示 CORS 错误

**解决方案**：
检查 `backend/.env` 文件中的 CORS 配置：
```env
CORS_ORIGINS=http://localhost:5173,http://localhost:3000
```

### 6. 前端无法连接后端

**问题**：`ERR_CONNECTION_REFUSED`

**解决方案**：
1. 确认后端服务已启动
2. 检查后端端口是否正确（默认 8000）
3. 检查防火墙设置
4. 查看 `frontend/.env` 中的 API 地址配置

## 📝 开发模式 vs 生产模式

### 开发模式

- 后端：使用 `--reload` 参数，代码修改自动重启
- 前端：使用 `npm run dev`，支持热更新
- 适合日常开发

### 生产模式

- 后端：不使用 `--reload`，性能更好
- 前端：使用 `npm run build` + `npm run preview`
- 适合部署

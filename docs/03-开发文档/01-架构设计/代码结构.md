# 代码结构文档（归类整理版）

本文档描述当前“离线单机版”系统的推荐代码组织结构、模块职责与边界，作为后续整理与扩展的基线。

## 1. 总体架构

- **前端**：`frontend/`（Vite + Vue 3 + Pinia + Element Plus）
- **后端**：`backend/`（FastAPI + SQLAlchemy + SQLite，离线单机模式）
- **容器化（可选）**：根目录 `docker-compose.yml`（包含 Redis 服务，但离线单机版后端默认使用本地缓存/SQLite）

> 说明：仓库内存在较多历史/实验/缓存目录（如多套 `venv_*`、`node_modules`、`__pycache__` 等）。本次已通过根目录 `.gitignore` 统一排除这些“运行产物”，避免继续污染仓库。

## 2. 后端目录与职责（`backend/app/`）

推荐按“分层 + 模块化路由”的方式组织，当前关键目录如下：

```
backend/app/
├── main.py                 # FastAPI 应用入口（lifespan、全局异常、路由注册、静态挂载）
├── api/                    # API 路由层（只做协议/校验/编排）
│   ├── __init__.py         # api_router 聚合
│   └── v1/                 # v1 版本路由
│       ├── auth.py         # 认证
│       ├── backup.py       # 备份
│       ├── organization.py # 组织管理
│       ├── policy.py       # 政策
│       └── village/        # 帮扶村模块（进一步模块化）
├── core/                   # 核心基础设施（配置、DB、缓存、安全、响应封装、指标）
├── middleware/             # 中间件（鉴权、RBAC、CORS 等）
├── models/                 # SQLAlchemy 模型（实体定义）
└── services/               # 业务服务（统计/分析/AI 等领域服务）
```

### 2.1 模块边界约定

- **`api/`**：不直接写复杂业务逻辑；只负责请求/响应协议、参数校验、调用 service、统一错误处理。
- **`services/`**：封装领域计算与业务编排；尽量不要依赖 FastAPI 对象（Request/Response），保持可测试。
- **`models/`**：只放 ORM 模型与基础关系/字段；避免把业务计算塞进模型方法里。
- **`core/`**：通用能力（配置、数据库、缓存、监控、安全、事务/响应工具）；对业务层保持“可复用、低耦合”。

## 3. 前端目录与职责（`frontend/src/`）

```
frontend/src/
├── main.ts                 # 应用入口（Pinia、Router、ElementPlus、全局样式）
├── router/                 # 路由 + 守卫 + 权限控制
├── store/                  # Pinia store 聚合
├── api/                    # API 封装（request.ts 及各业务模块）
├── layouts/                # 布局（DefaultLayoutSafe / LoginLayout）
├── views/                  # 页面级组件（按业务域分目录）
├── composables/            # 组合式工具（响应式侧边栏、加载、事件总线等）
└── styles/                 # 设计Token、主题与全局样式
```

### 3.1 UI 样式组织建议

- **设计变量统一入口**：`styles/tokens.scss`
- **全局样式入口**：`styles/index.scss`（只做 reset + 基础全局规则 + 引入 Token/布局样式）
- **组件/布局样式**：放在 `styles/components/*`，避免在大量 Vue SFC 内硬编码色值。

## 4. 仓库归类与清理规则

根目录 `.gitignore` 已统一忽略：

- Python：`__pycache__/`、`*.pyc`、`.pytest_cache/`、`.coverage` 等
- Node：`node_modules/`、`dist/`、`.cache/`、`nodejs/_cacache` 等
- 本地配置与密钥：`**/.env*`（保留 `*.env.example`）
- 离线数据：`backend/data/`、`backend/backups/`、`backend/uploads/`、`*.db`

> 后续如需“进一步归类整理”（例如把历史脚本、安装器、报告统一搬运到 `tools/`、`installer/`、`docs/`），建议按“先文档定规矩 → 再小步迁移 → 每步回归测试”的方式推进。

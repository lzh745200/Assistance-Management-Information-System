# 军队参与乡村振兴工作管理系统 - 核心框架体系

## 1. 技术栈选型

### 1.1 前端技术栈

- **核心框架**：Vue 3
- **构建工具**：Vite
- **路由管理**：Vue Router 4
- **状态管理**：Pinia
- **UI组件库**：Element Plus
- **HTTP客户端**：Axios
- **图表库**：ECharts
- **类型系统**：TypeScript
- **CSS预处理器**：SCSS

### 1.2 后端技术栈

- **核心框架**：FastAPI
- **数据库ORM**：SQLAlchemy
- **数据验证**：Pydantic
- **认证授权**：JWT (JSON Web Token)
- **数据库**：SQLite (开发/测试), PostgreSQL (生产)
- **缓存**：Redis
- **任务队列**：Celery (用于异步任务)
- **API文档**：Swagger/OpenAPI (FastAPI自带)

### 1.3 部署技术栈

- **容器化**：Docker
- **编排工具**：Docker Compose
- **CI/CD**：GitHub Actions / Jenkins
- **监控**：Prometheus + Grafana
- **日志管理**：ELK Stack

## 4. 接口设计规范

### 4.1 RESTful API 设计原则

#### 4.1.1 资源命名规范

- 使用名词表示资源
- 复数形式表示集合资源
- 嵌套资源使用斜杠分隔
- 示例：
- GET /api/v1/users - 获取用户列表
- GET /api/v1/users/{id} - 获取单个用户
- POST /api/v1/users - 创建用户
- PUT /api/v1/users/{id} - 更新用户
- DELETE /api/v1/users/{id} - 删除用户
- GET /api/v1/projects/{id}/participations - 获取项目参与人员

#### 4.1.2 HTTP方法使用

- GET: 获取资源
- POST: 创建资源
- PUT: 更新完整资源
- PATCH: 部分更新资源
- DELETE: 删除资源
- HEAD: 获取资源头信息

#### 4.1.3 状态码规范

- 200 OK: 成功获取资源
- 201 Created: 成功创建资源
- 204 No Content: 成功删除或更新资源
- 400 Bad Request: 请求参数错误
- 401 Unauthorized: 未授权
- 403 Forbidden: 禁止访问
- 404 Not Found: 资源不存在
- 500 Internal Server Error: 服务器内部错误

### 4.2 响应格式规范

#### 4.2.1 成功响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 具体数据
  },
  "timestamp": 1634567890
}
```text

#### 4.2.2 错误响应格式

```json
{
  "code": 400,
  "message": "错误信息",
  "error": {
    "detail": "详细错误描述",
    "fields": {
      "field_name": "字段错误信息"
    }
  },
  "timestamp": 1634567890
}
```text

#### 4.2.3 分页响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [
      // 数据列表
    ],
    "total": 100,
    "page": 1,
    "page_size": 20,
    "pages": 5
  },
  "timestamp": 1634567890
}
```text

### 4.3 请求参数规范

#### 4.3.1 查询参数

- 分页参数: page, page_size
- 排序参数: sort_by, sort_order (asc/desc)
- 过滤参数: 字段名=值
- 搜索参数: search
- 状态过滤: status=1,2,3

#### 4.3.2 请求头规范

- Content-Type: application/json
- Authorization: Bearer {token}
- X-Request-ID: 唯一请求标识
- X-User-ID: 用户ID

## 5. 依赖管理策略

### 5.1 前端依赖管理

#### 5.1.1 包管理工具

- 使用 npm 或 yarn 作为包管理工具
- 锁定依赖版本，避免意外更新
- 使用 package-lock.json 或 yarn.lock 确保依赖一致性

#### 5.1.2 依赖分类

- **核心依赖**: Vue, Vue Router, Pinia 等基础框架
- **UI组件**: Element Plus
- **功能库**: Axios, ECharts 等
- **开发依赖**: TypeScript, Vite, ESLint 等

#### 5.1.3 版本控制策略

- 核心框架使用语义化版本
- 关键依赖指定精确版本
- 定期更新依赖并进行兼容性测试

### 5.2 后端依赖管理

#### 5.2.1 包管理工具

- 使用 pip 作为包管理工具
- 使用 requirements.txt 锁定依赖版本
- 考虑使用 Poetry 进行高级依赖管理

#### 5.2.2 依赖分类

- **Web框架**: FastAPI
- **数据库**: SQLAlchemy, pymysql/postgresql
- **认证**: python-jose, passlib
- **数据验证**: Pydantic
- **工具库**: python-dotenv, python-multipart
- **测试工具**: pytest, httpx

#### 5.2.3 版本控制策略

- 生产环境使用固定版本
- 开发环境可以使用较新版本
- 使用虚拟环境隔离项目依赖

## 系统整体架构

本系统采用 **Electron 桌面壳 + 前后端分离** 的架构设计，以离线单机版形式交付，
支持 Windows 和麒麟 V10（x86_64 / ARM64）双平台部署。

```
+=====================================================================+
|                  Electron 28 桌面壳 (Desktop Shell)                  |
|  +---------------------------------------------------------------+  |
|  |  主进程 (Main Process)                                        |  |
|  |    - 窗口管理（无边框、自适应）                                |  |
|  |    - 后端进程管理（PyInstaller 打包的 FastAPI 子进程）         |  |
|  |    - 系统托盘（最小化、退出）                                  |  |
|  |    - 原生菜单与快捷键                                          |  |
|  +---------------------------------------------------------------+  |
+=====================================================================+
                              |
               +--------------+--------------+
               |                             |
+------------------------------+  +------------------------------+
|     前端层 (Frontend)        |  |      后端层 (Backend)        |
|  Vue 3.4 + TypeScript 5.9   |  |  Python 3.11 + FastAPI 0.109 |
|  Element Plus 2.13          |  |  SQLAlchemy 2.0 (ORM)        |
|  Pinia 2.1 (状态管理)        |  |  JWT + bcrypt (认证)         |
|  Vue Router 4.3             |  |  Pydantic 2.6 (验证)         |
|  ECharts 5.5 + Leaflet 1.9  |  |  diskcache (本地缓存)         |
|  Vite 7.3 (构建)             |  |  Alembic (数据库迁移)         |
|                              |  |                               |
|  87 页面 / 87 组件           |  |  657 API 端点 / 55 数据模型   |
+------------------------------+  +------------------------------+
               |                             |
               |                +------------------------------+
               |                |   DDD 分层架构                |
               |                |     API 路由层 (v1/)          |
               |                |     应用层 (application/)     |
               |                |     领域层 (domain/)          |
               |                |     基础设施层 (infra/)       |
               |                +------------------------------+
               |                             |
               +--------------+--------------+
                              |
+=====================================================================+
|                      数据层 (Data Layer)                             |
|  +----------------+  +----------------+  +------------------+       |
|  |   SQLite       |  |   文件存储     |  |    diskcache     |       |
|  |   (主数据库)    |  |   (上传文件)   |  |    (本地缓存)     |       |
|  |   55 张表       |  |   /uploads/    |  |    /data/cache/  |       |
|  +----------------+  +----------------+  +------------------+       |
+=====================================================================+
```

### DDD 分层架构详解

```
+------------------------------------------------------------------+
|                        API 路由层 (api/v1/)                       |
|  职责：HTTP 请求处理、参数验证、路由定义                           |
|  端点：657 个 REST API                                            |
+------------------------------------------------------------------+
           |                      |                      |
           v                      v                      v
+-------------------+  +-------------------+  +-------------------+
|   应用层          |  |   领域层          |  |   基础设施层      |
| (application/)    |  |   (domain/)       |  | (infrastructure/) |
|                   |  |                   |  |                   |
| 权限服务          |  | 领域模型          |  | diskcache 缓存    |
| 用例编排          |  | 领域事件          |  | 文件存储          |
| 事务管理          |  | 值对象            |  | 数据持久化        |
| 业务流程          |  | 业务规则          |  | 外部服务          |
+-------------------+  +-------------------+  +-------------------+
           |                      |                      |
           +----------------------+----------------------+
                                  |
                                  v
+------------------------------------------------------------------+
|                     数据持久化层 (models/)                         |
|  55 个 SQLAlchemy 模型文件 | 100+ 张数据库表                       |
+------------------------------------------------------------------+
```

### 核心特性

1. **API 设计**
   - RESTful API 设计，657 个端点
   - 自动生成 API 文档（Swagger UI / ReDoc）
   - 版本化 API（`/api/v1`）
   - 统一错误处理和响应格式
   - CSRF 防护（SameSite=Lax + Secure）

2. **数据处理**
   - SQLAlchemy 2.0 ORM（声明式映射）
   - Pydantic 2.6 数据验证
   - 事务管理（应用层统一管理）
   - 双轨数据库迁移（Alembic + 自动列补齐）

3. **安全机制**
   - JWT 认证（8 小时有效期）
   - bcrypt 密码加密（10 轮）
   - Token 黑名单（SQLite 存储）
   - RBAC + 数据级权限控制
   - 双因素认证支持
   - 请求速率限制
   - SQL 注入 / XSS / CSRF 防护

4. **缓存策略**
   - diskcache 本地磁盘缓存（100MB 上限）
   - 组织树结构缓存
   - 用户权限缓存
   - 统计数据缓存
   - 地图距离缓存（TTL=10min）
   - Dashboard 缓存（带失效机制）

5. **数据权限**
   - 四级数据范围：all / org_children / org / self
   - FK 精确过滤（替代文本模糊匹配）
   - 创建时自动继承组织 ID
   - 行级数据访问控制

### 核心特性

1. **组件化设计**
   - 基于 Element Plus 2.13 构建
   - 87 个可复用组件（24 个通用 + 63 个业务/模块组件）
   - 响应式布局，自适应窗口大小

2. **状态管理**
   - 18 个 Pinia Store，按模块划分
   - 类型安全的状态访问
   - 支持持久化缓存

3. **数据请求**
   - 统一的 Axios 请求封装（baseURL = `/api/v1`）
   - 请求/响应拦截器（Token 注入、错误处理）
   - 离线模式支持

4. **路由管理**
   - 90+ 路由，Vue Router 4.3
   - 路由懒加载（按需加载）
   - 路由守卫（认证、角色、权限）
   - 动态路由（根据角色显示菜单）

## 安全架构

### 认证与授权

```
+------------------+     +------------------+     +------------------+
|   JWT 认证       |     |   RBAC 权限      |     |   数据权限       |
|                  |     |                  |     |                  |
| 登录 → Token     |     | 角色 → 权限      |     | 数据范围过滤     |
| Token 黑名单     |     | 路由级验证       |     | all/org_children |
| 8h 有效期        |     | 资源级控制       |     | /org/self        |
| bcrypt 加密      |     | 7 张 RBAC 表     |     | FK 精确过滤      |
| 双因素认证       |     | 前端路由守卫     |     | 行级访问控制     |
+------------------+     +------------------+     +------------------+
```

### 安全防护

| 防护项 | 实现方式 |
|--------|----------|
| SQL 注入 | SQLAlchemy ORM 参数化查询 |
| XSS | DOMPurify 输入消毒 |
| CSRF | SameSite=Lax + Secure Cookie |
| 暴力破解 | 登录失败计数 + 账户锁定 |
| 数据加密 | AES-256 备份加密 |
| 审计追踪 | 全操作审计日志 |
| 会话管理 | Token 黑名单 + 会话超时 |

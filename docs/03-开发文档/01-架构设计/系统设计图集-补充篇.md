## 17. 错误处理与异常流程图

```mermaid
graph TB
    CLIENT["客户端请求"] --> MW["中间件链处理"]

    MW --> CATCH1{"anyio.EndOfStream?"}
    CATCH1 -->|"是 客户端断开"| R499["返回 499<br/>Client Closed Request"]
    CATCH1 -->|"否"| ROUTE["路由匹配"]

    ROUTE --> CATCH2{"路由存在?"}
    CATCH2 -->|"否"| R404["404 NotFound<br/>标准化错误响应"]

    CATCH2 -->|"是"| AUTH_CHECK["认证检查"]
    AUTH_CHECK --> CATCH3{"认证通过?"}
    CATCH3 -->|"否"| R401["401 Unauthorized"]

    CATCH3 -->|"是"| PERM_CHECK["权限检查"]
    PERM_CHECK --> CATCH4{"权限通过?"}
    CATCH4 -->|"否"| R403["403 Forbidden"]

    CATCH4 -->|"是"| VALIDATION["参数验证 Pydantic"]
    VALIDATION --> CATCH5{"验证通过?"}
    CATCH5 -->|"否"| R422["422 ValidationError<br/>详细字段错误"]

    CATCH5 -->|"是"| BUSINESS["业务逻辑处理"]
    BUSINESS --> CATCH6{"业务异常?"}
    CATCH6 -->|"是 AppError"| R400["400/409/...<br/>业务错误信息"]

    CATCH6 -->|"是 数据库异常"| DB_ERR["数据库错误处理<br/>回滚事务"]
    DB_ERR --> R500_DB["500 InternalError<br/>记录详细日志 exc_info=True"]

    CATCH6 -->|"是 未知异常"| UNKNOWN["未知异常捕获<br/>logger.error(exc_info=True)"]
    UNKNOWN --> R500["500 InternalError<br/>内部错误 不泄露细节"]

    CATCH6 -->|"否"| SUCCESS["200/201 成功响应<br/>{code, data, message}"]

    style R499 fill:#fff9c4,stroke:#f9a825
```

## 25. 数据包加密传输流程图

```mermaid
sequenceDiagram
    actor Sender as 发送方组织
    participant ExportSvc as 导出服务 (export.py)
    participant Encrypt as 加密模块 (cryptography)
    participant FileSystem as 文件系统
    participant Transfer as 传输媒介 (U盘/光盘)
    participant Receiver as 接收方组织
    participant ImportSvc as 导入服务 (import_data.py)

    Note over Sender,ImportSvc: ═══ 发送方: 数据导出与加密 ═══

    Sender->>ExportSvc: 选择导出数据范围
    ExportSvc->>ExportSvc: 查询数据 (应用 data_scope 过滤)
    ExportSvc->>ExportSvc: 序列化为 JSON 格式
    ExportSvc->>ExportSvc: 生成数据包元数据<br/>(组织/时间/版本/记录数)

    ExportSvc->>Encrypt: 请求加密
    Encrypt->>Encrypt: 生成随机 AES-256 密钥
    Encrypt->>Encrypt: AES-256-CBC 加密数据体
    Encrypt->>Encrypt: 生成 SHA-256 校验和
    Encrypt->>Encrypt: 用接收方公钥加密 AES 密钥
    Encrypt-->>ExportSvc: 加密后的数据包<br/>{ encrypted_key, iv, ciphertext, checksum }

    ExportSvc->>FileSystem: 写入 .mrrms-datapackage 文件
    FileSystem-->>Sender: 数据包文件就绪

    Sender->>Transfer: 拷贝到传输媒介

    Note over Sender,ImportSvc: ═══ 接收方: 数据解密与导入 ═══

    Transfer->>Receiver: 获取数据包文件
    Receiver->>ImportSvc: 上传数据包文件

    ImportSvc->>ImportSvc: 解析数据包元数据
    ImportSvc->>Encrypt: 请求解密
    Encrypt->>Encrypt: 用私钥解密 AES 密钥
    Encrypt->>Encrypt: AES-256-CBC 解密数据体
    Encrypt->>Encrypt: 验证 SHA-256 校验和

    alt 校验通过
        Encrypt-->>ImportSvc: 解密后的 JSON 数据
        ImportSvc->>ImportSvc: 数据校验 (Pydantic Schema)
        ImportSvc->>ImportSvc: 冲突检测 (唯一键/时间戳)
        ImportSvc->>ImportSvc: 合并策略: 保留最新 / 合并 / 手动

        opt 有冲突
            ImportSvc->>ImportSvc: 记录冲突到 data_conflicts 表
        end

        ImportSvc->>ImportSvc: 事务批量写入数据库
        ImportSvc-->>Receiver: 导入成功 { imported: N, conflicts: M }
    else 校验失败
        Encrypt-->>ImportSvc: 校验错误
        ImportSvc-->>Receiver: 导入失败: 数据完整性验证失败
    end
```

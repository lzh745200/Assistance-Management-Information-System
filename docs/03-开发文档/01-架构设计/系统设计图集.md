## 6. 认证与授权流程图

```mermaid
sequenceDiagram
    actor User as 用户
    participant Client as Vue 前端
    participant API as FastAPI 后端
    participant Auth as 认证模块 (security.py)
    participant DB as SQLite 数据库
    participant Cache as diskcache 缓存

    Note over User,Cache: ═══ 登录流程 ═══
    User->>Client: 输入用户名 / 密码
    Client->>API: POST /api/v1/auth/login
    API->>Auth: 验证凭据
    Auth->>DB: SELECT * FROM users WHERE username = ?
    DB-->>Auth: 用户记录 (含 bcrypt hash)
    Auth->>Auth: bcrypt 验证密码 (10 轮)
    Auth->>DB: 检查登录失败次数 / 锁定状态

    alt 验证成功
        Auth->>Auth: 生成 JWT Access Token (8h 有效期)
        Auth->>DB: 记录登录会话 (user_sessions)
        Auth->>Cache: 缓存用户权限列表
        Auth-->>API: { access_token, user }
        API-->>Client: 200 OK
        Client->>Client: Pinia auth store 存储 Token
        Client->>Client: 路由守卫 → 加载动态菜单
    else 验证失败
        Auth->>DB: INSERT INTO login_attempts (失败记录)
        Auth-->>API: 401 Unauthorized
        API-->>Client: { detail: "用户名或密码错误" }
    end

    Note over User,Cache: ═══ 后续请求认证链路 ═══
    Client->>API: GET /api/v1/villages (Bearer token)
    API->>Auth: JWT 中间件 → 解码 Token
    Auth->>Auth: 验证签名 (HS256) / 检查过期时间
    Auth->>DB: 检查 Token 是否在黑名单 (token_blacklist)
    Auth->>Cache: 获取用户角色 → 权限列表

    Note over User,Cache: ═══ RBAC 权限检查 ═══
    Auth->>Auth: 验证端点是否需要特殊权限

    Note over User,Cache: ═══ 数据权限过滤 (data_scope) ═══
    Auth->>Auth: 读取 user.data_scope 字段
    alt data_scope = "all" (管理员)
        Auth->>DB: SELECT ... (无过滤)
    else data_scope = "org_children" (管理岗)
        Auth->>DB: WHERE org_id IN (本组织及所有下级)
    else data_scope = "org" (审批岗)
        Auth->>DB: WHERE org_id = 用户组织 ID
    else data_scope = "self" (查看者)
        Auth->>DB: WHERE created_by = 用户 ID
    end
    DB-->>API: 返回过滤后的数据
    API-->>Client: 200 OK { code, data, message }
```

## 7. API 请求处理流程图

```mermaid
sequenceDiagram
    actor Client as Vue 3 前端
    participant Axios as Axios 请求封装
    participant CORS as CORS 中间件
    participant CSRF as CSRF 中间件
    participant Rate as 速率限制中间件
    participant Log as 审计日志中间件
    participant Router as FastAPI 路由匹配
    participant Dep as 依赖注入 (Depends)
    participant Schema as Pydantic 数据验证
    participant Svc as 业务服务层
    participant DB as SQLite 数据库
    participant Resp as 响应格式化

    Client->>Axios: request.get('/villages')
    Axios->>Axios: 注入 Authorization: Bearer token
    Axios->>Axios: baseURL = /api/v1 → 完整 URL

    Axios->>CORS: HTTP GET /api/v1/villages
    CORS->>CORS: 检查 Origin 头 / 添加 CORS 响应头
    CORS->>CSRF: 继续请求

    CSRF->>CSRF: 检查是否在 CSRF_EXEMPT_PATHS
    CSRF->>CSRF: 验证 SameSite=Lax Cookie
    CSRF->>Rate: 继续请求

    Rate->>Rate: 检查请求频率 / IP 限制
    Rate->>Log: 继续请求

    Log->>Log: 记录 API 访问日志 (api_access_logs)
    Log->>Router: 继续请求

    Router->>Router: 匹配路由 → /api/v1/villages
    Router->>Dep: 解析依赖注入链

    Dep->>Dep: get_db() → 数据库会话
    Dep->>Dep: get_current_user() → JWT 认证 + 权限
    Dep->>Dep: get_data_scope() → 数据范围过滤

    Dep->>Schema: 注入已验证的依赖到端点函数
    Schema->>Schema: Pydantic 验证查询参数 / 请求体
    Schema-->>Router: 验证通过

    Router->>Svc: 调用业务服务层
    Svc->>Svc: 业务逻辑处理 / 领域规则校验
    Svc->>DB: SQLAlchemy 参数化查询 (防注入)
    DB-->>Svc: 查询结果
    Svc-->>Router: 业务数据

    Router->>Resp: 格式化响应
    Resp->>Resp: 包装为 { code: 200, data: ..., message: "success" }
    Resp-->>Client: JSON Response 200 OK

    Note over Client,Resp: 异常: 404 NotFound | 422 ValidationError | 401 Unauthorized | 403 Forbidden | 500 InternalError
```

## 8. 审批工作流状态机

```mermaid
stateDiagram-v2
    [*] --> draft: 创建申请

    state draft {
        [*] --> editing: 编辑中
        editing --> editing: 保存草稿
        editing --> ready: 填写完成
    }

    draft --> submitted: 提交审批
    draft --> cancelled: 取消申请

    submitted --> pending_level1: 进入一级审批

    state pending_level1 {
        [*] --> waiting1: 等待审批人
        waiting1 --> reviewing1: 审批人查看
    }

    pending_level1 --> approved_level1: 一级审批通过
    pending_level1 --> rejected: 一级审批驳回
    pending_level1 --> returned_level1: 退回修改

    returned_level1 --> pending_level1: 修改后重新提交

    approved_level1 --> pending_level2: 进入二级审批
    approved_level1 --> approved: 仅需一级审批 (单级流程)

    pending_level2 --> approved_level2: 二级审批通过
    pending_level2 --> rejected: 二级审批驳回
    pending_level2 --> returned_level2: 退回修改

    returned_level2 --> pending_level2: 修改后重新提交

    approved_level2 --> pending_level3: 进入三级审批
    approved_level2 --> approved: 仅需二级审批

    returned_level3 --> pending_level3: 修改后重新提交

    end note

        - 通知服务 (消息推送)
```

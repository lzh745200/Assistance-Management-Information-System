## 1. 开发环境搭建

### 1.1 系统要求

- 操作系统：Windows 10/11、macOS 10.14+、Linux (Ubuntu 18.04+)
- Node.js：>=18.0.0
- npm：>=9.0.0
- Git：最新版本

### 1.2 环境配置步骤

#### 1.2.1 克隆项目

```bash
git clone https://github.com/your-org/military-rural-revitalization-system.git
cd military-rural-revitalization-system
```

#### 1.2.2 配置Node.js环境

```bash
# 安装前端依赖
cd frontend
npm install
```

### 1.3 开发工具推荐

- **IDE**：VS Code、WebStorm
- **API测试工具**：Postman、Insomnia
- **版本控制**：Git、GitHub Desktop

## 3. 开发流程

### 3.1 分支管理

项目采用Git Flow分支管理模型：

- **main**：主分支，用于生产环境
- **develop**：开发分支，用于集成新功能
- **feature/***：功能分支，用于开发新功能
- **release/***：发布分支，用于发布准备
- **hotfix/***：热修复分支，用于紧急修复

### 3.2 开发步骤

1. **创建功能分支**
   ```bash
   git checkout develop
   git pull origin develop
   git checkout -b feature/功能名称
   ```

2. **开发功能**
   - 编写代码
   - 添加测试
   - 更新文档

3. **提交代码**
   ```bash
   git add .
   git commit -m "feat: 添加新功能"
   git push origin feature/功能名称
   ```

4. **创建Pull Request**
   - 在GitHub上创建PR
   - 填写PR描述
   - 等待代码审查

5. **合并代码**
   - 通过代码审查后合并到develop分支
   - 定期将develop合并到main分支

## 4. 代码规范

- 遵循ESLint和Stylelint代码质量规范
- 使用TypeScript确保类型安全
- 组件名使用PascalCase
- 文件名使用kebab-case
- 变量和函数使用camelCase

示例：
```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

const getUserById = async (userId: number): Promise<User | null> => {
  try {
    const response = await api.get(`/users/${userId}`);
    return response.data;
  } catch (error) {
    console.error('获取用户失败:', error);
    return null;
  }
};
```

## 6. 测试指南

### 6.1 单元测试

```typescript
import { mount } from '@vue/test-utils';
import UserComponent from '@/components/UserComponent.vue';

describe('UserComponent', () => {
  it('renders user name', () => {
    const wrapper = mount(UserComponent, {
      props: { user: { name: 'John Doe' } }
    });
    expect(wrapper.text()).toContain('John Doe');
  });
});
```


## 7. 贡献流程

### 7.1 提交规范

提交信息遵循Conventional Commits规范：

- `feat:` 新功能
- `fix:` 修复bug
- `docs:` 文档更新
- `style:` 代码格式调整
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建过程或辅助工具的变动

示例：
```bash
git commit -m "feat: 添加用户管理功能"
git commit -m "fix: 修复登录验证问题"
```

### 7.2 代码审查

所有代码变更都需要通过代码审查：

1. 创建Pull Request
2. 填写PR描述
3. 等待团队成员审查
4. 根据反馈修改代码
5. 通过审查后合并代码

### 7.3 发布流程

1. 准备发布分支
2. 更新版本号
3. 更新CHANGELOG
4. 创建发布标签
5. 构建和发布

### 2.2 DDD 分层说明

#### API 层（app/api/）
- 处理 HTTP 请求和响应
- 路由定义和参数验证
- 调用应用层服务
- 不包含业务逻辑

#### 应用层（app/application/）
- 协调领域服务
- 编排业务流程
- 事务管理
- 不包含核心业务规则

#### 领域层（app/domain/）
- 核心业务逻辑
- 领域模型和规则
- 领域事件
- 与技术实现无关

#### 基础设施层（app/infrastructure/）
- 技术实现
- 数据持久化
- 缓存实现
- 外部服务集成

## 3. 开发流程

### 3.1 添加新的 API 接口

1. **定义数据模型**（如需要）
   ```python
   # app/models/example.py
   from sqlalchemy import Column, Integer, String
   from app.models.base import Base

   class Example(Base):
       __tablename__ = "examples"

       id = Column(Integer, primary_key=True, index=True)
       name = Column(String(100), nullable=False)
       description = Column(String(500))
   ```

2. **定义 Pydantic Schema**
   ```python
   # app/schemas/example.py
   from pydantic import BaseModel

   class ExampleCreate(BaseModel):
       name: str
       description: str | None = None

   class ExampleResponse(BaseModel):
       id: int
       name: str
       description: str | None

       class Config:
           from_attributes = True
   ```

3. **实现业务逻辑**
   ```python
   # app/application/services/example_service.py
   from sqlalchemy.orm import Session
   from app.models.example import Example
   from app.schemas.example import ExampleCreate

   class ExampleService:
       def create_example(self, db: Session, data: ExampleCreate) -> Example:
           example = Example(**data.dict())
           db.add(example)
           db.commit()
           db.refresh(example)
           return example
   ```

4. **添加 API 路由**
   ```python
   # app/api/v1/examples.py
   from fastapi import APIRouter, Depends
   from sqlalchemy.orm import Session
   from app.api.dependencies import get_db
   from app.application.services.example_service import ExampleService
   from app.schemas.example import ExampleCreate, ExampleResponse

   router = APIRouter(prefix="/examples", tags=["examples"])

   @router.post("/", response_model=ExampleResponse)
   def create_example(
       data: ExampleCreate,
       db: Session = Depends(get_db)
   ):
       service = ExampleService()
       return service.create_example(db, data)
   ```

5. **注册路由**
   ```python
   # app/api/v1/__init__.py
   from app.api.v1 import examples

   # 在 main.py 中注册
   app.include_router(examples.router, prefix="/api/v1")
   ```

6. **添加测试**
   ```python
   # tests/test_examples.py
   def test_create_example(client):
       response = client.post("/api/v1/examples/", json={
           "name": "Test",
           "description": "Test description"
       })
       assert response.status_code == 200
       assert response.json()["name"] == "Test"
   ```

## 4. 代码规范

### 4.1 Python 代码规范

- 遵循 **PEP 8** 代码风格
- 使用 **类型提示**（Type Hints）
- 函数和类添加 **文档字符串**（Google 风格）
- 变量和函数使用 **snake_case**
- 类名使用 **PascalCase**
- 常量使用 **UPPER_SNAKE_CASE**
- 最大行长度：**120 字符**

### 4.2 代码示例

```python
from typing import Optional
from sqlalchemy.orm import Session
from app.models.user import User

def get_user_by_id(db: Session, user_id: int) -> Optional[User]:
    """
    根据 ID 获取用户信息。

    Args:
        db: 数据库会话
        user_id: 用户 ID

    Returns:
        用户对象，如果不存在则返回 None
    """
    return db.query(User).filter(User.id == user_id).first()
```

## 7. 数据库迁移

### 7.1 双轨迁移机制

系统使用**双轨迁移机制**：

1. **自动列补齐**：启动时自动检测并添加缺失的列（适用于简单新增列）
2. **Alembic 迁移**：用于复杂 schema 变更（重命名/删除列、新建/删除表、索引变更等）

### 7.2 Alembic 命令

```bash
# 生成新迁移
alembic revision --autogenerate -m "描述变更内容"

# 执行迁移
alembic upgrade head

# 查看当前版本
alembic current

# 查看迁移历史
alembic history

# 回滚迁移
alembic downgrade -1  # 回滚一个版本
```

### 7.3 迁移最佳实践

1. **复杂变更必须使用 Alembic**，不要直接修改数据库
2. **生成迁移后检查脚本**，确保 DDL 正确
3. **测试迁移**，在开发环境先测试
4. **备份数据**，生产环境迁移前备份
5. **更新相关代码**，同步更新 models、schemas、services

## 8. 性能优化

### 8.1 数据库查询优化

```python
# 避免 N+1 查询
# 不好的做法
villages = db.query(Village).all()
for village in villages:
    projects = village.projects  # 每次都查询数据库

# 好的做法
from sqlalchemy.orm import joinedload
villages = db.query(Village).options(joinedload(Village.projects)).all()
```

### 9.2 数据库锁定

SQLite 在并发写入时可能锁定，解决方案：

```python
# 使用 WAL 模式
# 在 database.py 中配置
engine = create_engine(
    DATABASE_URL,
    connect_args={"check_same_thread": False, "timeout": 30}
)
```

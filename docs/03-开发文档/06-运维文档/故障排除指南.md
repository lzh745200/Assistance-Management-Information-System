# 故障排除指南

## 启动问题

### 1. 后端启动崩溃 - WinError 1392（文件或目录损坏）

**错误信息**:
```
OSError: [WinError 1392] 文件或目录损坏且无法读取。
  File "starlette\staticfiles.py", line 164, in lookup_path
    full_path = os.path.realpath(joined_path)
```

**根因**: NTFS 文件系统损坏，导致 `resources/frontend/assets/` 或 `frontend/node_modules/` 下的文件/目录条目损坏。

**诊断步骤**:

```bash
# 步骤1: 定位损坏文件/目录
python -c "
import os
path = '<报错路径>'
print('exists:', os.path.exists(path))
if os.path.isdir(path):
    try:
        print('listdir:', os.listdir(path))
    except OSError as e:
        print('CORRUPTED:', e)
"

# 步骤2: 扫描损坏范围
python -c "
import os
base = r'D:\military-Rural Revitalization-system'
# 检查关键目录
for d in ['resources/frontend/assets', 'frontend/node_modules']:
    path = os.path.join(base, d)
    if os.path.exists(path):
        try:
            # 递归遍历看哪些子目录无法访问
            for root, dirs, files in os.walk(path):
                pass
            print(f'OK: {d}')
        except OSError as e:
            print(f'CORRUPTED in {d}: {e}')
"
```

**修复流程**:

```bash
# 方案A: 绕过损坏目录（推荐，不需要 chkdsk /F）

# 1. 重命名损坏目录（mv 是元数据操作，不需要读取损坏内容）
mv resources/frontend resources/frontend_corrupted
mv frontend/node_modules frontend/node_modules_corrupted

# 2. 重建
cd frontend
npm install
npm run build
cp -r dist ../resources/frontend

# 3. 验证
cd ../backend
python start.py
# 确认无 "前端静态资源未挂载" 警告

# 4. 尝试清理损坏目录（可能需要先 chkdsk /F）
cmd /c "rmdir /s /q resources\frontend_corrupted"
cmd /c "rmdir /s /q frontend\node_modules_corrupted"
```

**根本修复**（需要管理员权限）:

```cmd
# 以管理员身份运行命令提示符，执行:
chkdsk D: /F

# 如果提示"是否计划在下一次系统重新启动时检查此卷"，输入 Y 并重启
```

### 2. 前端静态资源未挂载

**错误信息**:
```
WARNING - 前端静态资源未挂载。可能原因：
  1. 开发模式：请先执行 'cd frontend && npm run build' 构建前端
  2. 打包模式：确认 resources/frontend 目录存在且包含 index.html
```

**修复**:

```bash
# 查找前端目录的优先级：
# 1. 环境变量 FRONTEND_DIST_PATH
# 2. frontend/dist/
# 3. resources/frontend/

# 构建前端
cd frontend && npm run build

# 同步到 resources/
cp -r frontend/dist resources/frontend

# 或设置环境变量
export FRONTEND_DIST_PATH=$(pwd)/frontend/dist
```

### 3. 前端 JS/CSS 文件 404

**症状**: 后端正常启动，浏览器控制台出现:
```
GET /assets/js/index-XXXXX.js 404 Not Found
GET /assets/css/index-YYYYY.css 404 Not Found
```

**常见原因与修复**:

| 原因 | 修复 |
|------|------|
| **浏览器缓存**（最常见） | 硬刷新: `Ctrl+Shift+R` 或 `Ctrl+F5` |
| `resources/frontend/` 与 `frontend/dist/` 不同步 | `cp -r frontend/dist resources/frontend` |
| 构建不完整 | 重新 `cd frontend && npm run build` |

**验证一致性**:
```bash
python -c "
import hashlib
with open('frontend/dist/index.html', 'rb') as f:
    d = hashlib.md5(f.read()).hexdigest()
with open('resources/frontend/index.html', 'rb') as f:
    r = hashlib.md5(f.read()).hexdigest()
print('Match:', d == r)
"
```

## 构建问题

### 4. 前端构建失败 - node_modules 损坏

**错误示例**:
```
error during build:
node_modules/element-plus/es/components/select/src/select.mjs (1:77): 
  Identifier cannot follow number
```

**诊断**:
```bash
python -c "
path = 'frontend/node_modules/element-plus/es/components/select/src/select.mjs'
with open(path, 'r') as f:
    first_line = f.readline()
print('First 100 chars:', first_line[:100])
# 如果开头不是 'import' 或 '//' 而是乱码 → 文件损坏
"
```

**修复**:
```bash
# 重命名损坏的 node_modules
mv frontend/node_modules frontend/node_modules_corrupted

# 重新安装
cd frontend && npm install

# 如果 npm install 也失败（scandir 错误），说明 node_modules 中有目录级损坏
# 确保已成功重命名 node_modules 后再执行 npm install
```

### 5. npm install 失败 - scandir 错误

**错误示例**:
```
npm error code UNKNOWN
npm error syscall scandir
npm error path ...\node_modules\@antv\g6-plugin\tests\unit
npm error errno -4094
```

**修复**: 同问题 #4 的流程。`mv` 重命名操作是父子目录的元数据操作，不需要读取损坏子目录的内容。

## 数据库问题

### 6. 数据库完整性检查失败

**错误信息**:
```
[FAIL] 数据库完整性检查失败
```

**修复**:
```bash
# 恢复备份
cp backend/data/backup_*.db backend/data/military_rural.db

# 或重新初始化（会丢失数据）
rm backend/data/military_rural.db
cd backend && python start.py  # 自动创建新数据库
```

### 7. Alembic 迁移冲突

**症状**: 启动时 Schema 迁移报错。

**修复**:
```bash
cd backend
# 查看当前迁移状态
alembic current

# 标记为最新（跳过有问题的迁移）
alembic stamp head

# 重新生成迁移
alembic revision --autogenerate -m "fix_schema"
alembic upgrade head
```

## 系统级问题

### 8. 端口被占用

**错误**: `[Errno 10048] 通常每个套接字地址(协议/网络地址/端口)只允许使用一次`

**修复**:
```bash
# Windows - 查找占用端口的进程
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# 或修改配置文件中的端口号
```

### 9. Python 依赖缺失

**症状**: `ModuleNotFoundError` 或 `ImportError`

**修复**:
```bash
cd backend
pip install -r requirements.txt

# 如果特定包安装失败
pip install <package_name> --force-reinstall
```

### 10. VC++ 运行时缺失

**警告**:
```
[WARN] 缺少 VC++ 运行时 DLL: vcruntime140_1.dll
```

**修复**: 下载安装 [Visual C++ 2015-2022 Redistributable (x64)](https://aka.ms/vs/17/release/vc_redist.x64.exe)

## 快速诊断脚本

将以下内容保存为 `diagnose.py` 并在项目根目录运行：

```python
"""系统快速诊断脚本"""
import os, sys

BASE = os.path.dirname(os.path.abspath(__file__))

def check_dir(path, name):
    """检查目录是否存在且可访问"""
    full = os.path.join(BASE, path)
    if not os.path.exists(full):
        return f'[MISS] {name}: {path}'
    try:
        if os.path.isdir(full):
            items = os.listdir(full)
            return f'[OK]   {name}: {path} ({len(items)} items)'
        else:
            return f'[OK]   {name}: {path} (file)'
    except OSError as e:
        return f'[CORRUPTED] {name}: {path} ({e})'

def main():
    print('=' * 50)
    print('系统诊断')
    print('=' * 50)

    checks = [
        ('backend/start.py', '后端入口'),
        ('frontend/package.json', '前端配置'),
        ('frontend/node_modules', '前端依赖'),
        ('frontend/dist/index.html', '前端构建产物'),
        ('resources/frontend/index.html', '生产静态资源'),
        ('backend/data', '数据库目录'),
    ]

    for path, name in checks:
        print(check_dir(path, name))

    # 检查前端一致性
    dist_idx = os.path.join(BASE, 'frontend/dist/index.html')
    res_idx = os.path.join(BASE, 'resources/frontend/index.html')
    if os.path.isfile(dist_idx) and os.path.isfile(res_idx):
        import hashlib
        with open(dist_idx, 'rb') as f:
            d = hashlib.md5(f.read()).hexdigest()
        with open(res_idx, 'rb') as f:
            r = hashlib.md5(f.read()).hexdigest()
        status = '[OK]' if d == r else '[MISMATCH]'
        print(f'{status} 前端 dist ↔ resources 一致性')

if __name__ == '__main__':
    main()
```

# 帮扶管理信息系统 - 麒麟 V10 离线部署完整方案

> **适用场景**：目标麒麟 V10 x86_64 机器完全离线，且没有任何 Linux 构建环境
> **文档版本**：v1.1.0
> **更新日期**：2026-04-21

---

## 重要前提说明

**.deb 安装包中的后端程序 `assistance-management-backend` 是 Linux ELF 可执行文件，必须在 Linux 环境下编译。**

Windows 上的 PyInstaller 只能生成 `.exe`，无法生成 Linux 可执行文件。因此，要获得完整的 `.deb` 安装包，**必须获得一个 Linux 环境**。

本文档提供 5 种方案，按推荐程度排序。请根据你的实际情况选择。

---

## 快速决策表

| 方案 | 前提条件 | 难度 | 耗时 | 推荐度 | 适用情况 |
|------|---------|------|------|--------|---------|
| **方案一：WSL2** | Windows 能临时联网1小时 | 低 | 30分钟 | ⭐⭐⭐⭐⭐ | 首选，最省事 |
| **方案二：联网Linux机器** | 有/能借到联网Linux | 最低 | 15分钟 | ⭐⭐⭐⭐⭐ | 有现成Linux环境 |
| **方案三：VMware+ISO** | 能从别处下载ISO文件 | 中 | 2小时 | ⭐⭐⭐⭐ | 完全不能联网 |

| **方案五：交叉构建** | 熟悉代码修改 | 高 | 半天 | ⭐⭐ | 特殊需求 |

# 方案一：Windows 安装 WSL2 构建（最推荐）

> **前提**：你的 Windows 电脑可以临时联网（哪怕只有1小时，用手机热点也行）
>
> **原理**：WSL2（Windows Subsystem for Linux）是 Windows 内置的 Linux 子系统，无需虚拟机，性能接近原生 Linux

---

## 第一阶段：安装 WSL2（约15分钟，需联网）

### 步骤 1.1：检查 Windows 版本

按 `Win + R`，输入 `winver`，回车。

**要求**：Windows 10 版本 2004 及更高版本（内部版本 19041 及更高版本）或 Windows 11。

如果版本太低，请先到 Windows Update 更新系统。

### 步骤 1.2：以管理员身份打开 PowerShell

点击"开始"菜单 → 搜索 `PowerShell` → 右键点击"Windows PowerShell" → 选择"以管理员身份运行"。

**确认**：PowerShell 窗口标题栏应显示"管理员"。

### 步骤 1.3：启用 WSL 功能

在 PowerShell 中粘贴并执行以下命令（逐行执行）：

```powershell
# 启用 WSL（Windows Linux 子系统）
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
```

等待显示"操作成功完成"。

```powershell
# 启用虚拟机平台（WSL2 需要此功能）
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

等待显示"操作成功完成"。

> **解释**：这两行命令是开启 Windows 的两个系统功能。`dism.exe` 是 Windows 的部署映像服务和管理工具，`/enable-feature` 用于启用系统功能，`/norestart` 表示不立即重启。

### 步骤 1.4：设置 WSL 默认版本为 2

```powershell
wsl --set-default-version 2
```

**可能出现的情况**：
- 如果显示"有关 WSL2 的详细信息，请访问 https://aka.ms/wsl2"，说明设置成功，继续下一步。
- 如果提示"WSL 2 需要更新其内核组件"，请执行下面的步骤 1.5。
- 如果提示"无法识别 wsl"，说明步骤 1.3 没有成功，请重新执行。

### 步骤 1.5：更新 WSL2 内核（如需要）

如果步骤 1.4 提示需要更新内核，执行：

```powershell
# 下载 WSL2 内核更新包（约 14MB）
Invoke-WebRequest -Uri https://wslstorestorage.blob.core.windows.net/wslblob/wsl_update_x64.msi -OutFile wsl_update_x64.msi

# 静默安装
msiexec /i wsl_update_x64.msi /quiet
```

> **解释**：`Invoke-WebRequest` 是 PowerShell 的下载命令，`-Uri` 指定下载地址，`-OutFile` 指定保存文件名。`msiexec /i` 是安装 MSI 安装包，`/quiet` 表示静默安装（不显示界面）。

### 步骤 1.6：重启电脑

```powershell
Restart-Computer
```

> **必须重启**，WSL2 的虚拟化功能需要重启后才能生效。

## 第二阶段：安装 Ubuntu 22.04（约15分钟，需联网）

### 步骤 2.1：再次以管理员打开 PowerShell

重启完成后，重复步骤 1.2 的操作。

### 步骤 2.2：安装 Ubuntu 22.04

```powershell
wsl --install -d Ubuntu-22.04
```

**预期输出**：
```
正在安装: Ubuntu 22.04 LTS
[======================] 安装进度 100%
正在启动 Ubuntu 22.04 LTS...
```

> **解释**：这条命令会从 Microsoft Store 下载 Ubuntu 22.04（约 500MB）。下载完成后会自动启动 Ubuntu。

**如果下载很慢或失败**，可以尝试先安装 Ubuntu 再设为 WSL2：
```powershell
wsl --install -d Ubuntu-22.04 --web-download
```

### 步骤 2.3：配置 Ubuntu 用户名和密码

安装完成后，会自动弹出一个 Ubuntu 终端窗口，显示：

```
Installing, this may take a few minutes...
Please create a default UNIX user account. The username does not need to match your Windows username.
For more information visit: https://aka.ms/wslusers
Enter new UNIX username:
```

**输入用户名**（建议用简单好记的，如 `build`）：
```
build
```

然后设置密码：
```
New password:          ← 输入密码（输入时不显示任何字符，这是正常的）
Retype new password:   ← 再次输入确认
```

> **重要**：请记住这个用户名和密码！后续所有 Ubuntu 操作都需要用到。

**验证安装成功**：
```bash
whoami
```

应输出你刚才设置的用户名（如 `build`）。

---

## 第三阶段：在 Ubuntu 中安装构建环境（约10分钟，需联网）

保持 Ubuntu 终端窗口打开，逐行执行以下命令。

### 步骤 3.1：更新软件源

```bash
sudo apt-get update
```

> **解释**：`apt-get` 是 Ubuntu/Debian 系统的包管理器，`update` 是更新软件包列表，确保后续安装的包是最新版本。

**输入密码**：提示 `[sudo] password for build:` 时，输入你刚才设置的密码（输入时不显示字符）。

### 步骤 3.2：安装 Node.js 18

```bash
# 下载 NodeSource 的安装脚本
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
```

> **解释**：`curl` 是命令行下载工具，`-fsSL` 表示静默下载跟随重定向。这条命令下载 Node.js 18 的软件源配置脚本并执行。

```bash
# 安装 Node.js 和 npm
sudo apt-get install -y nodejs
```

> **解释**：`-y` 表示自动回答"是"，不需要手动确认。

**验证 Node.js 安装**：
```bash
node -v
```

**预期输出**：`v18.x.x`（如 `v18.20.4`）

```bash
npm -v
```

**预期输出**：`9.x.x` 或 `10.x.x`

### 步骤 3.3：安装 Python3 和构建工具

```bash
sudo apt-get install -y python3 python3-venv python3-pip build-essential dpkg-dev apt-utils
```

> **解释**：
> - `python3`：Python 解释器
> - `python3-venv`：创建虚拟环境的工具
> - `python3-pip`：Python 包管理器
> - `build-essential`：包含 gcc/g++ 等编译工具（某些 Python 包需要编译）
> - `dpkg-dev`：构建 .deb 包的工具
> - `apt-utils`：apt 工具集

**验证 Python 安装**：
```bash
python3 --version
```

**预期输出**：`Python 3.10.x`

**验证 PyInstaller 后续可用**：
```bash
which python3
```

应输出类似 `/usr/bin/python3` 的路径。

## 第四阶段：构建 DEB 安装包（约20-30分钟，需联网一次）

> **说明**：从现在开始，如果网络断开，只要前三个阶段已完成，后续构建命令依然可以执行（因为依赖已经下载到本地）。

### 步骤 4.1：进入项目目录

```bash
cd /mnt/c/military-Rural-Revitalization-system
```

> **解释**：WSL2 会自动挂载 Windows 的所有磁盘。`C:` 盘挂载在 `/mnt/c`，所以 Windows 上的 `C:\military-Rural-Revitalization-system` 在 Ubuntu 中的路径就是 `/mnt/c/military-Rural-Revitalization-system`。

**验证路径正确**：
```bash
pwd
```

应输出：`/mnt/c/military-Rural-Revitalization-system`

```bash
ls -la
```

应看到 `backend`、`frontend`、`scripts`、`package.json` 等文件和文件夹。

### 步骤 4.2：安装项目根目录的 npm 依赖

```bash
npm install
```

> **解释**：这会读取项目根目录的 `package.json`，安装 Electron、electron-builder 等工具。需要下载约 200MB 的文件，耗时 3-5 分钟。

**预期输出**：
```
added xxx packages in xxs
```

**如果卡住不动**：
- 可能是网络问题，按 `Ctrl+C` 取消后重试
- 或者更换 npm 镜像源：
  ```bash
  npm config set registry https://registry.npmmirror.com
  npm install
  ```

### 步骤 4.3：安装前端依赖

```bash
cd frontend
npm install
cd ..
```

> **解释**：前端代码在 `frontend` 文件夹里，有自己的 `package.json`。`cd ..` 是返回上级目录（回到项目根目录）。

**预期输出**：
```
added xxx packages in xxs
```

### 步骤 4.4：创建 Python 虚拟环境并安装依赖

```bash
cd backend
python3 -m venv .venv
```

> **解释**：`python3 -m venv .venv` 创建一个名为 `.venv` 的虚拟环境。虚拟环境的作用是隔离项目依赖，避免和系统 Python 冲突。

**激活虚拟环境并安装依赖**：
```bash
.venv/bin/pip install --upgrade pip
.venv/bin/pip install -r requirements.txt pyinstaller
```

> **解释**：
> - `.venv/bin/pip`：使用虚拟环境里的 pip（不是系统的 pip）
> - `--upgrade pip`：把 pip 本身升级到最新版
> - `-r requirements.txt`：按照清单安装所有 Python 依赖
> - `pyinstaller`：额外安装打包工具

**这个过程需要 5-10 分钟**，会看到大量下载进度条。

**验证 PyInstaller 安装成功**：
```bash
.venv/bin/python -c "import PyInstaller; print(PyInstaller.__version__)"
```

**预期输出**：`6.x.x`（版本号）

返回项目根目录：
```bash
cd ..
```

### 步骤 4.5：执行构建

```bash
bash scripts/build/build-installer-kylin.sh --arch x64
```

> **解释**：`bash` 执行 shell 脚本，`--arch x64` 指定构建 x86_64 架构的版本（适合你的麒麟 V10）。

**构建过程分 5 个阶段**，你会看到类似输出：

## 第五阶段：获取产物并拷贝到目标麒麟机器（离线操作）

### 步骤 5.1：在 Windows 中找到产物

构建完成后，在 Windows 文件资源管理器中打开：

```
C:\military-Rural-Revitalization-system\dist\electron\
```

你应该能看到：
```
assistance-management-system_1.1.0_amd64.deb    （约 350-400MB）
```

### 步骤 5.2：拷贝到 U 盘

把这个 `.deb` 文件复制到 U 盘。

### 步骤 5.3：在目标离线麒麟机器上安装

把 U 盘插入目标麒麟 V10 机器，打开终端，执行：

```bash
# 进入 U 盘挂载目录（通常是 /media/用户名/U盘名）
cd /media/$(whoami)/*/

# 或手动找到 .deb 文件位置
cd /media/$(whoami)/你的U盘名/

# 安装
sudo dpkg -i assistance-management-system_1.1.0_amd64.deb
```

> **解释**：`dpkg -i` = Debian Package Install，安装 .deb 包。

**如果提示缺少依赖**（通常麒麟 V10 已预装大部分库）：
```bash
sudo apt-get install -f
```

> **解释**：`-f` = fix broken，自动修复缺失的依赖。

### 步骤 5.4：启动软件

安装完成后，在麒麟的应用菜单中搜索"帮扶管理信息"，点击图标启动。

或者在终端中启动：
```bash
/opt/帮扶管理信息系统/帮扶管理信息系统
```

## 方案一常见问题排查

### Q1：`wsl --install` 提示"无法解析服务器的名称或地址"

**原因**：电脑没有联网。

**解决**：确保电脑能访问互联网。如果公司网络有限制，尝试手机热点。

### Q2：Ubuntu 安装完成后无法输入密码

**原因**：Linux 终端输入密码时**不显示任何字符**（包括星号），这是正常的安全设计。

**解决**：直接输入密码后按回车即可。

### Q3：`npm install` 报错 "ECONNRESET" 或 "ETIMEDOUT"

**原因**：网络不稳定，下载中断。

**解决**：
```bash
# 清除 npm 缓存后重试
npm cache clean --force
npm install
```

或者更换国内镜像源：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### Q4：`pyinstaller` 打包时提示 "cannot find libxxx"

**原因**：缺少某些系统库。

**解决**：
```bash
sudo apt-get install -y libffi-dev libssl-dev
```

然后重新运行构建脚本。

### Q5：构建完成后找不到 `.deb` 文件

**排查步骤**：
```bash
# 在 Ubuntu 中查找
find /mnt/c/military-Rural-Revitalization-system/dist -name "*.deb"

# 或在 Windows 中查找
dir "C:\military-Rural-Revitalization-system\dist\electron\*.deb" /s
```

如果还是找不到，说明构建失败，请查看构建日志的最后 50 行：
```bash
cd /mnt/c/military-Rural-Revitalization-system
bash scripts/build/build-installer-kylin.sh --arch x64 2>&1 | tail -50
```

# 方案二：找一台联网的 Linux 机器构建（最简单）

> **前提**：你能找到一台可以联网的电脑，安装了 Ubuntu、Debian、麒麟或其他 Linux 发行版
>
> **原理**：直接在 Linux 上执行构建，无需任何虚拟化或子系统

---

## 准备工作

### 需要准备的物品

1. 一个 U 盘（容量大于 4GB，用于拷贝项目代码和最终产物）
2. 一台联网的 Linux 机器（可以是：
   - 同事的 Ubuntu/麒麟电脑
   - 实验室的 Linux 服务器
   - 租用的云服务器（阿里云/腾讯云/华为云，最低配即可，约 0.1元/小时）
   - 树莓派或其他 ARM 设备（注意：如果是 ARM 设备，需要使用 `--arch arm64`）
   ）

### 在联网 Linux 机器上检查环境

打开终端，执行：

```bash
# 检查 Node.js
node -v

# 检查 Python3
python3 --version

# 检查 npm
npm -v

# 检查是否有 sudo 权限
sudo whoami    # 应输出 root
```

如果缺少任何一项，参考下面的安装命令。

## 第二阶段：拷贝项目代码到联网 Linux 机器

### 方式 A：用 U 盘拷贝

1. 在你的 Windows 电脑上，把项目文件夹复制到 U 盘
2. 把 U 盘插入联网 Linux 机器
3. 在 Linux 上挂载 U 盘（通常自动挂载到 `/media/用户名/U盘名`）

```bash
# 查看 U 盘挂载位置
ls /media/$(whoami)/

# 拷贝项目到用户目录
cp -r /media/$(whoami)/U盘名/military-Rural-Revitalization-system ~/
```

### 方式 B：用 `scp` 命令从 Windows 发送到 Linux（两台电脑在同一局域网）

在 Windows 的 PowerShell 或 CMD 中执行：

```powershell
# 格式：scp -r 本地路径 用户名@LinuxIP:远程路径
scp -r C:\military-Rural-Revitalization-system build@192.168.1.100:~/
```

然后输入 Linux 机器的密码。

### 方式 C：如果是云服务器，用 `scp` 或 FTP 上传

```bash
# 从本地上传到云服务器
scp -r ~/military-Rural-Revitalization-system root@你的服务器IP:/root/
```

## 第四阶段：拷贝产物到目标麒麟机器

### 步骤 4.1：确认产物存在


# 方案四：不构建 .deb → 在麒麟 V10 上直接源码运行（最务实）

> **前提**：你完全不想/不能获得 Linux 构建环境来打包 .deb，只想让系统在麒麟上跑起来
>
> **原理**：不把后端打包成单文件，而是直接在麒麟上安装 Python3，运行后端源码。前端用浏览器直接访问，不用 Electron 壳。
>
> **优点**：不需要 PyInstaller、不需要 electron-builder、不需要构建 .deb、不需要 Linux 构建环境
>
> **缺点**：部署步骤较多，目标机器上需要安装 Python 依赖

## 核心思路

```
Windows（构建前端）──U盘──→ 麒麟 V10（运行后端 + 浏览器访问前端）
```

具体流程：
1. 在 Windows 上编译前端（生成静态网页文件）
2. 在 Windows 上准备 Linux 版本的 Python 依赖包
3. 把所有文件用 U 盘拷贝到麒麟机器
4. 在麒麟上安装 Python3、创建虚拟环境、安装依赖
5. 启动后端服务，用浏览器访问

## 第一阶段：在 Windows 上准备前端（离线可完成）

### 步骤 1.1：构建前端

在 Windows 终端中，进入项目根目录：

```bash
cd C:\military-Rural-Revitalization-system\frontend
npm run build
```

> **解释**：`npm run build` 把 Vue 源代码编译成浏览器能直接运行的静态文件，输出到 `frontend/dist` 目录。

**验证构建成功**：
```

├── index.html          ← 有这个文件说明成功
├── assets/
└── ...
```

### 步骤 1.2：准备 Linux 版本的 Python 依赖包

这是整个方案中最关键也最复杂的一步。由于你的 Windows 是离线的，你需要从别处获取 Linux 版本的 Python wheel 包。

#### 方式 A：找一台联网的电脑下载 wheel 包（推荐）

在任意联网的 Linux 电脑上执行：

```bash
# 创建下载目录
mkdir -p ~/rural_deps && cd ~/rural_deps

# 下载项目 requirements.txt 中所有依赖的 Linux x86_64 版本
pip download \
  -r /path/to/military-Rural-Revitalization-system/backend/requirements.txt \
  --platform manylinux2014_x86_64 \
  --python-version 310 \
  --implementation cp \
  --only-binary=:all: \
  --no-deps \
  -d ./wheels
```

> **参数解释**：
> - `--platform manylinux2014_x86_64`：下载 Linux x86_64 兼容版本
> - `--python-version 310`：Python 3.10
> - `--implementation cp`：CPython 实现
> - `--only-binary=:all:`：只下载二进制 wheel 包，不下载源码包（避免在目标机器上编译）
> - `--no-deps`：不下载依赖的依赖（先尝试，如果失败再去掉这个参数）
> - `-d ./wheels`：保存到 wheels 目录

**下载完成后**，把 `wheels` 文件夹拷贝到 U 盘。

#### 方式 B：如果找不到联网 Linux，用 pip 的离线缓存

在联网的 Windows 上（如果同事/朋友电脑能联网）：

```powershell
# 安装相同版本的 Python 3.10
# 下载所有依赖（虽然下载的是 Windows 版本，但可以用作参考）
pip download -r requirements.txt -d ./wheels
```

> **注意**：Windows 下载的 wheel 包后缀是 `win_amd64`，不能直接用于 Linux。你需要额外下载 Linux 版本。

#### 方式 C：直接在目标麒麟上在线安装（如果麒麟能临时联网）

如果目标麒麟机器可以临时联网，最简单的做法是在麒麟上直接 pip install：

```bash
# 在麒麟上执行
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

安装完成后断开网络即可。

### 步骤 1.3：准备启动脚本

在 Windows 上，用记事本创建一个 `start-offline.sh` 文件，内容如下：

```bash
#!/bin/bash
# ============================================================
# 帮扶管理信息系统 - 麒麟 V10 离线启动脚本
# ============================================================

# 获取脚本所在目录
APP_DIR="$(cd "$(dirname "$0")" && pwd)"
cd "$APP_DIR"

echo "=========================================="
echo "  帮扶管理信息系统 - 启动中"
echo "=========================================="

# 检查 Python3
if ! command -v python3 &> /dev/null; then
    echo "[错误] 未找到 python3，请先安装 Python3"
    exit 1
fi

# 检查虚拟环境
if [ ! -d "$APP_DIR/backend/.venv" ]; then
    echo "[错误] 未找到虚拟环境，请先执行安装脚本"
    exit 1
fi

# 激活虚拟环境
source "$APP_DIR/backend/.venv/bin/activate"

# 确保运行时目录存在
mkdir -p "$APP_DIR/data"
mkdir -p "$APP_DIR/logs"
mkdir -p "$APP_DIR/uploads"
mkdir -p "$APP_DIR/exports"
mkdir -p "$APP_DIR/cache"
mkdir -p "$APP_DIR/backups"

# 设置环境变量
export DATABASE_URL="sqlite:///./data/rural_revitalization.db"
export UPLOAD_DIR="./uploads"
export EXPORT_DIR="./exports"
export CACHE_DIR="./data/cache"
export LOG_FILE="./logs/app.log"
export BACKUP_DIR="./backups"

# 如果环境变量中没有密钥，自动生成
if [ -z "$SECRET_KEY" ]; then
    export SECRET_KEY="$(python3 -c 'import secrets; print(secrets.token_urlsafe(48))')"
fi
if [ -z "$CSRF_SECRET_KEY" ]; then
    export CSRF_SECRET_KEY="$(python3 -c 'import secrets; print(secrets.token_urlsafe(48))')"
fi

# 启动后端服务
echo "[1/2] 正在启动后端服务..."
cd "$APP_DIR/backend"
python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 &
BACKEND_PID=$!
cd "$APP_DIR"

# 等待后端启动
sleep 5

# 检查后端是否启动成功
if ! curl -s http://localhost:8000/health &> /dev/null; then
    echo "[警告] 后端服务可能未正常启动，等待更长时间..."
    sleep 5
fi

echo "[2/2] 后端服务已启动: http://localhost:8000"
echo ""
echo "请用浏览器打开以下地址访问系统："
echo "  http://localhost:8000"
echo ""
echo "或打开前端静态页面（如果前端已构建）："
echo "  file://$APP_DIR/frontend/dist/index.html"
echo ""
echo "按 Ctrl+C 停止服务"

trap "echo ''; echo '正在停止服务...'; kill $BACKEND_PID 2>/dev/null; exit 0" INT TERM

# 等待用户按 Ctrl+C
wait
```

保存为 UTF-8 编码格式，文件名：`start-offline.sh`

## 方案四常见问题排查

### Q1：麒麟上 Python 版本太低（3.7 或更低）

**解决**：尝试从源码编译安装 Python 3.10（需要较多时间和磁盘空间）：

```bash
# 下载 Python 3.10 源码（从别处拷贝到 U 盘）
cd /tmp
tar -xzf Python-3.10.12.tgz
cd Python-3.10.12

# 编译安装
sudo apt-get install -y build-essential libssl-dev zlib1g-dev
./configure --prefix=/usr/local/python3.10
make -j$(nproc)
sudo make install

# 创建符号链接
sudo ln -s /usr/local/python3.10/bin/python3.10 /usr/local/bin/python3.10
sudo ln -s /usr/local/python3.10/bin/pip3.10 /usr/local/bin/pip3.10

# 使用新版本创建虚拟环境
python3.10 -m venv /opt/assistance-management-system/backend/.venv
```

### Q2：`pip install` 提示"No matching distribution found"

**原因**：wheel 包中的某些依赖版本与 requirements.txt 不匹配。

**解决**：
```bash
# 先安装能装上的
pip install --no-index --find-links ./wheels $(ls wheels/*.whl | xargs -n1 basename | sed 's/-[^-]*-[^-]*-[^-]*\.whl//g' | sort -u)

# 然后查看还缺什么
pip install -r backend/requirements.txt --dry-run
```

### Q3：启动后端时提示"ModuleNotFoundError"

**原因**：某些依赖没有正确安装。

**解决**：
```bash
cd /opt/assistance-management-system/backend
source .venv/bin/activate

# 手动安装缺失的模块
pip install xxx

# 或者重新安装所有依赖
pip install --no-index --find-links ../wheels -r requirements.txt
```

### Q4：浏览器访问 localhost:8000 显示"无法连接"

**排查步骤**：
```bash
# 检查后端进程是否在运行
ps aux | grep uvicorn

# 检查端口是否监听
netstat -tlnp | grep 8000

# 或
curl http://localhost:8000/health
```

### Q5：前端页面打开后显示白屏或报错

**原因**：前端通过 `file://` 协议打开，存在 CORS 限制。

**解决**：确保通过 `http://localhost:8000` 访问，让后端提供前端静态文件。或者修改后端配置支持 CORS：

在 `backend/app/main.py` 中找到 CORS 配置，确保包含：
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 允许所有来源
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

# 方案五：Windows 交叉构建（高级）

> **前提**：你有丰富的 Node.js/Electron 开发经验，愿意修改代码
>
> **原理**：electron-builder 支持在 Windows 上构建 Linux 目标，但只能打包 Electron 壳和前端。后端 Python 程序需要单独处理。
>
> **适用场景**：你希望得到一个"看起来"是 .deb 的安装包，但实际上里面装的是 Windows 构建的前端 + 麒麟上源码运行的后端。

---

## 核心问题

**electron-builder 在 Windows 上构建 `--linux deb` 时**：
- ✅ 可以打包 Electron 主进程和前端渲染进程
- ✅ 可以打包 `extraResources` 中指定的额外文件
- ❌ 不能执行 Linux 特有的 PyInstaller 打包
- ❌ 生成的后端可执行文件是 Windows 的 `.exe`，不是 Linux 的 ELF

**因此需要变通方案**：
1. Windows 上构建一个"精简版"的 .deb（只包含前端和 Electron 壳）
2. 在麒麟上单独安装 Python3 和后端源码
3. 修改 Electron 的 main.js，让它在 Linux 上启动 Python 后端而不是找可执行文件

---

## 第一阶段：在 Windows 上构建前端 .deb

### 步骤 1.1：修改 package.json

编辑项目根目录的 `package.json`，修改 `build.linux` 部分：

```json

    }

}
```


> **解释**：electron-builder 内置了跨平台构建能力。在 Windows 上指定 `--linux deb`，它会使用预编译的 Linux 二进制工具和 fakeroot 来模拟 Linux 环境生成 .deb 包。

**可能遇到的问题**：
- 如果提示缺少 `wine` 或相关工具，这是正常的，因为某些 electron-builder 功能需要 Wine。但纯 .deb 构建通常不需要 Wine。
- 如果构建失败，可能需要安装 Windows 版的 fakeroot 等工具。

## 第二阶段：修改 Electron 主进程支持 Linux 源码运行

### 步骤 2.1：修改 electron/main.js

编辑 `electron/main.js`，添加 Linux 平台的特殊处理：

```javascript
const { app, BrowserWindow } = require('electron');
const path = require('path');
const { spawn } = require('child_process');

let backendProcess = null;

function startBackend() {
  const isLinux = process.platform === 'linux';

  if (isLinux) {
    // Linux 平台：启动 Python 源码
    const backendDir = path.join(process.resourcesPath, '..', '..', 'backend');
    const venvPython = path.join(backendDir, '.venv', 'bin', 'python3');

    console.log('Linux 平台：启动 Python 后端...');
    console.log('Python 路径:', venvPython);

    backendProcess = spawn(venvPython, ['-m', 'uvicorn', 'app.main:app', '--host', '0.0.0.0', '--port', '8000'], {
      cwd: backendDir,
      env: {
        ...process.env,
        DATABASE_URL: 'sqlite:///./data/rural_revitalization.db',
      }
    });

    backendProcess.stdout.on('data', (data) => {
      console.log(`[后端] ${data}`);
    });

    backendProcess.stderr.on('data', (data) => {
      console.error(`[后端错误] ${data}`);
    });

    backendProcess.on('close', (code) => {
      console.log(`后端进程退出，代码: ${code}`);
    });

    // 等待后端启动
    return new Promise((resolve) => {
      setTimeout(() => {
        resolve('http://localhost:8000');
      }, 5000);
    });
  } else {
    // Windows 平台：启动打包后的可执行文件
    const backendExe = path.join(process.resourcesPath, 'backend', 'assistance-management-backend.exe');
    // ... 原有 Windows 启动逻辑 ...
  }
}

async function createWindow() {
  const backendUrl = await startBackend();

  const mainWindow = new BrowserWindow({
    width: 1400,
    height: 900,
    webPreferences: {
      nodeIntegration: false,
      contextIsolation: true,
      preload: path.join(__dirname, 'preload.js')
    }
  });

  // 加载后端提供的前端页面
  mainWindow.loadURL(backendUrl);
}

app.whenReady().then(createWindow);

app.on('window-all-closed', () => {
  if (backendProcess) {
    backendProcess.kill();
  }
  if (process.platform !== 'darwin') {
    app.quit();
  }
});
```

### 步骤 2.2：修改 package.json 的 extraResources

让 .deb 包包含后端源码：

```json
"extraResources": [
  {
    "from": "frontend/dist",
    "to": "frontend"
  },
  {
    "from": "backend",
    "to": "backend",
    "filter": [
      "app/**/*",
      "alembic/**/*",
      "alembic.ini",
      ".env",
      "requirements.txt",
      "start.py"
    ]
  }
]
```

### 步骤 2.3：修改安装后脚本

编辑 `build/linux/after-install.sh`，添加 Python 环境初始化：


## 方案五的局限性

1. **安装时间较长**：安装 .deb 后还需要在目标机器上 pip install，可能需要 5-10 分钟
2. **依赖风险**：如果某些 Python 包需要编译（没有 manylinux wheel），安装会失败
3. **体积较大**：.deb 包需要包含后端源码（而不是单个可执行文件），体积会增加
4. **首次启动慢**：首次启动需要初始化数据库和加载依赖

**因此，方案五只推荐给有 Electron 开发经验的用户，或者实在无法获得 Linux 构建环境的特殊情况。**

# 部署检查清单

## 1. 环境准备验证

### 1.1 服务器环境检查

- [ ] 确认操作系统版本符合要求 (Linux CentOS 8+/Ubuntu 20.04+ 或 Windows Server 2019+)
- [ ] 验证Python版本 (3.9+)
- [ ] 验证Node.js版本 (16.x LTS 或 18.x LTS)
- [ ] 确认数据库服务可用 (PostgreSQL 13+)
- [ ] 确认缓存服务可用 (Redis 6+)
- [ ] 验证必要的系统工具已安装 (Git, Docker, Docker Compose)
- [ ] 确认防火墙配置正确，必要端口已开放
- [ ] 验证服务器磁盘空间充足 (至少10GB可用空间)
- [ ] 确认服务器内存满足要求 (至少4GB RAM)

### 1.2 环境变量配置

- [ ] 验证所有必需的环境变量已正确设置
- [ ] 确认数据库连接字符串格式正确
- [ ] 验证API密钥和敏感配置已安全存储
- [ ] 检查环境配置文件权限是否正确 (防止未授权访问)
- [ ] 确认不同环境(开发、测试、生产)的配置隔离

## 2. 代码与配置检查

### 2.1 代码质量检查

- [ ] 运行代码风格检查 (Python: black, flake8; JavaScript: eslint)
- [ ] 验证代码格式是否符合项目规范
- [ ] 检查是否存在未使用的导入或变量
- [ ] 验证所有函数和类都有适当的文档字符串/注释
- [ ] 确认代码没有明显的逻辑错误或安全漏洞
- [ ] 验证代码遵循项目的命名约定
- [ ] 检查是否存在硬编码的敏感信息

### 2.2 配置文件验证

- [ ] 验证配置文件格式正确 (YAML/JSON/INI)
- [ ] 确认所有必需的配置项都已设置
- [ ] 验证配置值的合理性 (端口号、超时时间等)
- [ ] 检查配置文件是否包含环境特定的配置
- [ ] 确认日志配置正确设置

## 3. 依赖与资源验证

### 3.1 依赖完整性验证

- [ ] 确认所有依赖都已正确安装 (运行依赖版本跟踪工具)
- [ ] 验证依赖版本与文档中指定的版本一致
- [ ] 检查是否存在依赖冲突
- [ ] 确认生产环境中没有包含开发依赖
- [ ] 验证依赖的许可证兼容性

### 3.2 资源文件检查

- [ ] 确认所有必要的静态资源都已包含
- [ ] 验证前端构建产物完整性
- [ ] 检查数据库迁移文件是否最新
- [ ] 确认所有模板文件都已正确渲染
- [ ] 验证图片、字体等资源文件完整性

## 4. 安全与合规检查

### 4.1 安全漏洞扫描

- [ ] 运行依赖安全扫描 (Python: safety, npm audit)
- [ ] 验证没有已知的高危安全漏洞
- [ ] 检查是否存在过时的库版本
- [ ] 确认所有依赖的CVSS评分都在可接受范围内
- [ ] 审查安全扫描报告，确认已解决关键和高危漏洞

### 4.2 代码安全检查

- [ ] 验证认证和授权机制正常工作
- [ ] 确认所有输入都经过适当的验证和清理
- [ ] 检查是否存在SQL注入、XSS等常见安全漏洞
- [ ] 验证敏感数据传输是否使用HTTPS/TLS加密
- [ ] 确认敏感数据存储是否加密
- [ ] 检查是否实施了适当的速率限制
- [ ] 验证日志记录不会泄露敏感信息

## 5. 部署后验证步骤

### 5.1 功能验证

- [ ] 确认所有API端点都能正常响应
- [ ] 验证关键业务流程是否正常工作
- [ ] 检查用户认证和授权是否正常
- [ ] 测试数据增删改查操作
- [ ] 验证前端与后端交互是否正常
- [ ] 确认所有第三方集成服务正常工作

### 5.2 性能验证

- [ ] 验证API响应时间符合性能要求
- [ ] 检查系统在负载下的稳定性
- [ ] 确认数据库查询性能正常
- [ ] 验证缓存机制正常工作
- [ ] 检查服务器资源使用率是否在合理范围内

### 5.3 监控与日志验证

- [ ] 确认日志记录正常工作
- [ ] 验证监控系统能够捕获关键指标
- [ ] 检查告警机制是否配置正确
- [ ] 确认错误日志包含足够的诊断信息
- [ ] 验证审计日志记录了必要的操作

## 6. 回滚准备

### 6.1 回滚计划验证

- [ ] 确认存在详细的回滚计划文档
- [ ] 验证数据库备份是否可用且最新
- [ ] 确认代码版本控制正确配置
- [ ] 检查是否有回滚所需的所有配置文件备份
- [ ] 验证回滚脚本是否可执行

### 6.2 团队确认

- [ ] 确认所有团队成员了解部署计划和回滚程序
- [ ] 验证部署时间窗口已安排且通知相关人员
- [ ] 确认部署责任人已明确
- [ ] 检查是否有足够的支持人员待命

## 7. 部署工具和自动化

### 7.1 CI/CD配置检查

- [ ] 确认CI/CD流水线配置正确
- [ ] 验证自动化测试在CI/CD中执行
- [ ] 检查部署脚本是否包含所有必要步骤
- [ ] 确认自动化部署过程中的错误处理

### 7.2 部署后清理

- [ ] 确认临时文件已清理
- [ ] 验证日志轮转配置正确
- [ ] 检查不再需要的资源是否已释放

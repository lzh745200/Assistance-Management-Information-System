## 快速开始

### 1. 安装测试依赖

```cmd
cd backend
.venv\Scripts\activate
pip install pytest pytest-html pytest-cov psutil faker playwright locust
playwright install
```

### 2. 运行所有测试

**方法 A：使用批处理脚本（推荐）**

```cmd
cd C:\military-Rural Revitalization-system
run_tests.bat
```

**方法 B：使用 pytest**

```cmd
cd backend
pytest ../tests/ -v --html=../test_report.html
```

### 3. 查看测试报告

打开生成的 `test_report.html` 文件查看详细测试结果。

## 测试分类

### 按测试类型运行

```cmd
# 数据安全测试
pytest tests/security/test_data_security.py -v

# 安全测试
pytest tests/security/test_security.py -v

# UI/UX 测试
pytest tests/ui-ux/test_ui_ux.py -v

# 稳定性测试（快速）
pytest tests/stability/test_stability.py -v -m "not slow and not stress"

# E2E 测试
pytest tests/e2e/test_e2e.py -v
```

### 按测试标记运行

```cmd
# 只运行快速测试
pytest tests/ -v -m "not slow and not stress"

# 运行长稳测试
pytest tests/stability/ -v -m slow

# 运行压力测试
pytest tests/stability/ -v -m stress

# 运行安全测试
pytest tests/ -v -m security

# 运行集成测试
pytest tests/ -v -m integration
```

### 按测试类运行

```cmd
# 运行特定测试类
pytest tests/security/test_data_security.py::TestDataEncryption -v

# 运行特定测试方法
pytest tests/security/test_data_security.py::TestDataEncryption::test_database_file_not_plaintext -v
```

## 测试工具

### 1. 测试数据生成器

生成测试所需的模拟数据。

```cmd
cd backend
python ../tests/tools/test_data_generator.py

# 清理测试数据
python ../tests/tools/test_data_generator.py --clear

# 自定义数量
python ../tests/tools/test_data_generator.py --users 20 --villages 50 --projects 100
```

### 2. 性能测试（Locust）

启动性能测试工具。

```cmd
# 启动 Locust Web 界面
locust -f tests/performance/locustfile.py --host=http://localhost:8000

# 无界面模式
locust -f tests/performance/locustfile.py --host=http://localhost:8000 --headless -u 100 -r 10 -t 5m
```

参数说明：
- `-u 100`：模拟 100 个用户
- `-r 10`：每秒增加 10 个用户
- `-t 5m`：运行 5 分钟

### 3. E2E 测试（Playwright）

运行浏览器自动化测试。

```cmd
# 运行 E2E 测试
pytest tests/e2e/ -v

# 有头模式（显示浏览器）
pytest tests/e2e/ -v --headed

# 慢速模式（便于观察）
pytest tests/e2e/ -v --headed --slowmo 1000

# 指定浏览器
pytest tests/e2e/ -v --browser chromium
pytest tests/e2e/ -v --browser firefox
pytest tests/e2e/ -v --browser webkit
```

### 4. 测试覆盖率

生成代码覆盖率报告。

```cmd
# 运行测试并生成覆盖率报告
pytest tests/ --cov=app --cov-report=html --cov-report=term

# 查看覆盖率报告
start htmlcov/index.html
```

## CI/CD 集成

### GitHub Actions

项目已配置 GitHub Actions 自动化测试流程（`.github/workflows/tests.yml`）。

**触发条件：**
- Push 到 master/main/develop 分支
- 创建 Pull Request

**测试流程：**
1. 后端测试（数据安全、安全性、稳定性）
2. 前端测试（类型检查、Lint、单元测试）
3. E2E 测试（端到端测试）
4. 安全扫描（Bandit、Safety）

**查看测试结果：**
- 访问 GitHub 仓库的 Actions 标签页
- 查看测试报告 Artifacts

## 测试最佳实践

### 1. 测试前准备

```cmd
# 1. 确保虚拟环境激活
cd backend
.venv\Scripts\activate

# 2. 更新测试依赖
pip install -r requirements-dev.txt

# 3. 生成测试数据
python ../tests/tools/test_data_generator.py

# 4. 启动服务（E2E 测试需要）
python start.py
```

### 2. 测试命名规范

- 测试文件：`test_*.py`
- 测试类：`Test*`
- 测试方法：`test_*`

### 3. 测试标记使用

```python
import pytest

@pytest.mark.slow
def test_long_running():
    """长时间运行的测试"""
    pass

@pytest.mark.stress
def test_high_load():
    """压力测试"""
    pass

@pytest.mark.integration
def test_api_integration():
    """集成测试"""
    pass
```

### 4. 测试数据隔离

```python
@pytest.fixture
def test_db():
    """测试数据库 fixture"""
    # 创建测试数据库
    setup_test_database()
    yield
    # 清理测试数据库
    teardown_test_database()
```

## 常见问题

### Q1: 测试失败怎么办？

**A:** 查看详细错误信息：

```cmd
pytest tests/ -v --tb=long
```

### Q2: 如何跳过某些测试？

**A:** 使用 `-k` 参数：

```cmd
# 跳过包含 "slow" 的测试
pytest tests/ -v -k "not slow"

# 只运行包含 "security" 的测试
pytest tests/ -v -k "security"
```

### Q3: 如何调试测试？

**A:** 使用 `--pdb` 参数：

```cmd
pytest tests/security/test_security.py -v --pdb
```

### Q4: E2E 测试失败？

**A:** 检查以下几点：

1. 后端服务是否启动（http://localhost:8000）
2. 前端服务是否启动（http://localhost:5173）
3. Playwright 浏览器是否安装（`playwright install`）

### Q5: 性能测试如何设置目标？

**A:** 编辑 `tests/performance/locustfile.py`，调整用户数和请求频率。

### Q6: 如何生成测试报告？

**A:** 使用 pytest-html 插件：

```cmd
pytest tests/ -v --html=report.html --self-contained-html
```

### Q7: 测试覆盖率太低怎么办？

**A:**

1. 查看覆盖率报告找出未覆盖的代码
2. 为未覆盖的代码编写测试用例
3. 运行覆盖率测试验证

```cmd
pytest tests/ --cov=app --cov-report=html
start htmlcov/index.html
```

### Q8: 如何并行运行测试？

**A:** 安装 pytest-xdist：

```cmd
pip install pytest-xdist
pytest tests/ -v -n auto  # 自动检测 CPU 核心数
```

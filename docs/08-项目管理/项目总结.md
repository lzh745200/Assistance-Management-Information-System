## 版本历史

### v1.1.0 (2026-04-14)

**主题：数据权限体系完善 + 系统优化**

本版本重点完善了数据权限系统，实现了基于 RBAC 的精细化数据访问控制，同时进行了全面的系统性能优化和代码质量提升。

#### 新增功能

- **数据权限系统（Data Permission）**
  - 实现基于角色的数据范围控制（RBAC + Data Scope）
  - 支持四种数据范围级别：全部数据（`all`）、本组织及下级（`org_children`）、仅本组织（`org`）、仅本人（`self`）
  - 新增 `DataScope` 依赖项，提供 `filter_by_org_ids()` 和 `require_data_permission()` 方法
  - 数据权限覆盖所有核心业务模块：帮扶村、帮扶学校、项目、资金

- **用户管理增强**
  - 新增管理员密码重置接口 `POST /users/{id}/admin-reset-password`
  - 用户管理页面完整重写：支持角色分配、数据范围设置、权限管理
  - 新增待审核用户管理功能

- **资金生命周期管理**
  - 新增资金审批工作流端点：approve / reject / allocate / start_use / complete / audit
  - 资金表新增 `organization_id` 和 `created_by` 字段，支持数据权限过滤
  - 数据库迁移 `79113cc0494a` 添加资金权限字段

#### 改进优化

- **Dashboard 优化**
  - 使用 SQL 子查询替代 Python list 物化，提升查询性能
  - 审批动态显示实际用户名（`joinedload(ApprovalTask.submitter)`）
  - Dashboard 缓存失效机制完善，数据变更时自动刷新

- **缓存系统优化**
  - `cache.py` / `config.py` / `backup_service.py` 路径集中化
  - 修复缓存 TTL 配置 bug
  - 地图距离查询增加 diskcache 缓存（TTL=10 分钟），坐标更新时自动清除

- **代码质量提升**
  - 移除死代码：`comments.py`（依赖不存在的 collaboration 模型）、`unique_validation.py`（路由前缀冲突）
  - 移除废弃的 `filter_query` 方法（`data_scope.py`）
  - 统一响应判断工具函数
  - 前端 API 路径规范化，消除 `/api/v1` 重复拼接问题
  - Flake8 0 错误，TypeScript 0 错误

- **登录响应解析修复**
  - 修复前端登录响应解析问题，统一响应判断工具

#### Bug 修复

- 修复 CSRF cookie `SameSite=Lax` + `Secure` 逻辑
- 修复 `metrics.py` 延迟加载问题，添加 `prometheus_client` 数据收集
- 修复 `UserManagement.vue` 中 `name` / `full_name` 字段不一致问题
- 修复待审核用户列表双重 fetch 问题
- 修复 `admin_reset_password` 接口参数类型（`dict` -> `AdminResetPasswordBody(BaseModel)`）
- 修复 `get_pending_users` 中 `== False` 改为 `.is_(False)`（SQLAlchemy 规范）

#### 数据库变更

| 迁移版本 | 内容 |
|----------|------|
| `79113cc0494a` | funds 表添加 `organization_id` 和 `created_by` 字段 |

## 项目概述

军队乡村振兴管理系统是一个基于 Electron 的离线单机版桌面应用，用于管理军队参与乡村振兴的全流程数据。系统采用前后端分离架构，通过 Electron 打包为桌面应用，支持完全离线运行，满足无网络环境下的办公需求。

### 项目基本信息

| 项目 | 信息 |
|------|------|
| 项目名称 | 军队乡村振兴管理系统 |
| 项目类型 | 离线单机版桌面应用 |
| 当前版本 | 1.1.1 |
| 技术栈 | Python + FastAPI + Vue 3 + Electron |
| 数据库 | SQLite（单文件） |
| 部署方式 | NSIS 安装包 / DEB 包 / Portable |

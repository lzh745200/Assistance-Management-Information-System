# v1.4.0 Phase 1 设计文档

> **日期**: 2026-06-06 | **状态**: 已批准 | **下一阶段**: 实施计划

## 概述

v1.4.0 是"数据协同 + 业务流程完善"的第一个子版本，聚焦三个核心能力：

1. **数据协同冲突检测** — 多机数据包导入时的冲突检测与手动裁决
2. **自动化提醒引擎** — 审批超时、项目到期、经费告警的主动通知
3. **流程串联** — 经费↔项目双向联动

## 一、数据协同冲突检测引擎

### 架构

```
分支机构数据包(.rrs)
    ↓
导入校验器（格式校验 + 版本检查）
    ↓
冲突检测引擎（3 层规则）
    ├─ 无冲突 → 直接写入数据库
    └─ 有冲突 → 手动裁决界面 → 写入数据库
```

### 冲突检测规则

| 规则 | 检测条件 | 处理 |
|------|---------|------|
| ID 冲突 | 同一 ID 的实体在导入包和目标库中都存在 | 字段级对比，标黄差异 |
| 唯一键冲突 | 帮扶村名称+县区重复 | 标记为疑似重复 |
| 外键完整性 | 包中引用的 FK 在目标库中不存在 | 标记为孤儿记录 |

### 冲突解决策略（D 方案）

- 无冲突 → 自动追加（字段值全相同或包中数据是新增的）
- 时间戳冲突 → 默认取最新版本，但允许手动覆盖
- 手动裁决 → 并排对比界面，逐条选择"保留本地"或"采用导入"

## 二、自动化提醒引擎

### 提醒类型

| 类型 | 触发条件 | 通知方式 |
|------|---------|---------|
| 审批超时 | 审批任务 pending 超过 48h | 系统消息 + 红色标记 |
| 项目到期 | 结束日期前 7 天 / 已过期 | 系统消息 + 黄/红标记 |
| 经费告警 | 使用率 ≥ 80% (黄) / ≥ 95% (红) | 系统消息 + 顶部横幅 |
| 数据过期 | 年度数据超过 12 个月未更新 | 静默标记，列表下调 |
| 待办提醒 | 有未完成待办 | 登录后弹窗（每日一次） |

### 架构

```
后台定时扫描（5 分钟间隔）
    ├─ 审批表：status=pending AND elapsed > 48h → Message(type=overtime)
    ├─ 项目表：end_date - now ≤ 7d → Message(type=deadline)
    ├─ 经费表：used/approved ≥ 0.8 → Message(type=budget_warning)
    └─ 年度数据表：MAX(year) < 今年-1 → staleness flag
                    ↓
        前端顶部横幅 + 消息中心红点计数
```

## 三、经费↔项目双向联动

### 经费→项目（向上联动）

```
经费审批通过
    ↓
检测 project_id 是否为空
    ├─ 已关联 → 项目预算自动更新
    └─ 未关联 → 详情页提示"建议关联到项目" + 快捷入口
```

### 项目→经费（向下联动）

```
创建项目
    ↓
查询该项目所属帮扶村的可关联经费
    ├─ 有 → 列表展示，用户勾选
    └─ 无 → 提示"建议先申请经费"
```

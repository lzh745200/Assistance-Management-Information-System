## 二、已修复问题（10项）

### P0-1：transaction.py SET TRANSACTION 语法 SQLite 不支持（必崩）
- **文件**：`backend/app/core/transaction.py:233-237`
- **问题**：`_apply_tx_settings` 执行 `SET TRANSACTION ISOLATION LEVEL ...` 和 `SET TRANSACTION READ ONLY`，这是 MySQL/PostgreSQL 专有语法，SQLite 直接抛 `OperationalError`
- **影响**：任何使用 `@with_transaction(isolation_level=...)` 的路由在 SQLite 下必崩
- **修复**：增加 `if IS_SQLITE: return` 短路，SQLite 隔离级别通过 PRAGMA 控制
- **测试**：3 个断言 SET TRANSACTION 被调用的测试更新为按 IS_SQLITE 分支断言

### P0-2：machine_code.py 500错误泄露内部信息（安全）
- **文件**：`backend/app/api/v1/machine_code.py`（16处）
- **问题**：`HTTPException(status_code=500, detail=f"...失败: {str(e)}")` 将异常堆栈/SQL错误暴露给客户端
- **修复**：批量移除 `str(e)`，改为 `"...失败，请稍后重试或联系管理员"`，异常详情仅记录在服务端日志

### P0-3：reports.py 500错误泄露内部信息（安全）
- **文件**：`backend/app/api/v1/data/data/reports.py`（17处）
- **问题**：同 P0-2，`detail=f"...失败：{str(e)}"` 泄露内部错误
- **修复**：批量移除 `str(e)`，3处无 logger 的补加 `logger.error`

### P1-4：get_db 异常时缺显式 rollback
- **文件**：`backend/app/core/database.py:123-129`
- **问题**：`get_db` 仅 `db.close()`，未在异常分支显式 `db.rollback()`，连接池复用可能携带脏事务状态
- **修复**：改为 `try: yield; except: db.rollback(); raise; finally: db.close()`

### P1-5：engine 配置项未生效
- **文件**：`backend/app/core/database.py:55-63`
- **问题**：`pool_pre_ping=True` 和 `pool_recycle=3600` 硬编码，`DB_POOL_TIMEOUT`/`DB_ECHO` 配置项完全未传入
- **修复**：统一从 settings 读取：`pool_pre_ping=getattr(settings,"DB_POOL_PRE_PING",True)` 等

### P1-6：Village.organization_id 缺 ondelete 策略
- **文件**：`backend/app/models/village.py:48`
- **问题**：`ForeignKey("organizations.id")` 无 `ondelete`（默认 RESTRICT），删除组织时村庄会阻塞
- **修复**：改为 `ForeignKey("organizations.id", ondelete="SET NULL")`

### P1-7：Organization.created_by 缺索引
- **文件**：`backend/app/models/organization.py:36-42`
- **问题**：`created_by` 字段无索引，按创建人查询组织全表扫描
- **修复**：`__table_args__` 增加 `Index("ix_organizations_created_by", "created_by")`

### P1-8：auth/users.py except: pass 吞错误
- **文件**：`backend/app/api/v1/auth/users.py:767-768`
- **问题**：密码修改审计日志写入失败时 `except Exception: pass`，无任何日志记录
- **修复**：改为 `except Exception as audit_err: logger.warning(...)`，并补加 logger 导入

### 测试修复-9：test_core_transaction.py 断言过时
- **文件**：`backend/tests/unit/test_core_transaction.py`
- **问题**：3个测试断言 `SET TRANSACTION` 被调用，但 P0-1 修复后 SQLite 下已短路
- **修复**：更新为按 `IS_SQLITE` 分支断言

### 测试修复-10：test_data_reports_api_2.py CSRF 拦截（预存bug）
- **文件**：`backend/tests/unit/test_data_reports_api_2.py`
- **问题**：`mock_settings` fixture 漏设 `CSRF_ENABLED=False`，覆盖了 conftest 的全局 autouse fixture，导致 26 个 POST 测试被 CSRF 中间件 403 拦截
- **修复**：补设 `os.environ["CSRF_ENABLED"]="false"` + `settings.CSRF_ENABLED=False`

## 3. TypeScript 类型错误 (444)

**状态: ⚠️ 已知遗留, 不在本次 P0-P3 修复范围**

### 3.1 错误分布

| 错误码 | 数量 | 说明 |
|--------|------|------|
| TS2345 | ~120 | DefaultRow 不匹配具体类型 (Element Plus table 行类型推断问题) |
| TS2305 | ~80 | 模块无导出成员 (API 与 types 不同步) |
| TS2322 | ~60 | 类型不兼容 |
| TS2339 | ~50 | 属性不存在 |
| TS7006 | ~30 | 隐式 any |
| TS2554 | ~25 | 参数数量不匹配 |
| 其他 | ~80 | 各种 |

### 3.2 已知根因 (非修复目标)
- `el-table` 行类型自动推导为 `DefaultRow`, 与业务类型 (如 `SupportedVillage`) 不匹配
- 部分 stores (menu, rbac) 缺少 `canAccessMenu`、`hasRole` 等方法
- `policy.ts` 引用了不存在的 `Policy`、`PolicyQuery` 等 API 导出
- `permissionGuard.ts` 多次调用函数时参数数量不匹配

### 3.3 修复路径 (后续工作)
1. 在 `el-table` 上显式声明 `data: SupportedVillage[]`
2. 补全 stores 缺失方法 (menu, rbac)
3. 同步 `api/policy.ts` 与 `types/policy.ts`
4. 修正 `permissionGuard.ts` 的函数签名

**预计工作量: 3-5 天**

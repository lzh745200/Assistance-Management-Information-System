### 1.1 ai_service 多维度分析框架扩展

| 属性 | 值 |
|------|----|
| 优先级 | P1 |
| 预计工时 | 16h |
| 风险 | 低 |
| 涉及文件 | `backend/app/services/ai_service.py` |

**现状**：`AIServiceManager` 提供 6 个独立分析方法（`analyze_income_trend` / `analyze_project_progress` / `analyze_fund_efficiency` / `compare_villages` / `forecast_income_trend` / `forecast_fund_completion`），通过 `analyze_data(data, analysis_type)` 单入口分发，但**无统一聚合出口**，前端仪表盘需多次往返调用。对比维度仅按 `county` 聚合，风险评级阈值（0.9 / 0.7）硬编码。

**改进措施**：

1. 新增综合分析聚合出口 `analyze_comprehensive(db)`，一次调用返回多维仪表盘：

```python
def analyze_comprehensive(self, db: Session) -> Dict[str, Any]:

    return {
        "income_trend": self.analyze_income_trend(db),
        "project_progress": self.analyze_project_progress(db),
        "fund_efficiency": self.analyze_fund_efficiency(db),
        "fund_risk": self.forecast_fund_completion(db),
        "generated_at": datetime.now().isoformat(),
    }
```

2. 扩展对比维度：`compare_villages` 现仅按 `county` 聚合，增加 `support_unit`（帮扶单位）/ `department`（部门）维度参数。

3. 风险评级阈值外置到 `Settings` 配置：

```python
# backend/app/core/config.py
FUND_HIGH_RISK_THRESHOLD: float = 0.9
FUND_MEDIUM_RISK_THRESHOLD: float = 0.7
```

**验收**：新增 `/ai/comprehensive` 端点返回 4 维聚合数据；阈值可通过环境变量调整。

### 1.2 nlp_query 正则模板增强

| 属性 | 值 |
|------|----|
| 优先级 | P1（v1.0 为 P0，因 bug 已修复降级） |
| 预计工时 | 10h |
| 风险 | 中（涉及查询逻辑，需回归测试） |
| 涉及文件 | `backend/app/services/ai/nlp_query_service.py` |

**现状**：模板库仅 7 个（`village_by_province` / `village_count` / `project_count` / `project_by_status` / `total_funds` / `village_income` / `top_villages_by_income`），缺少"某村项目进度""某单位资金使用率""逾期项目清单"等军审高频问句。匹配策略为"首个命中即返回"，多模板命中无置信度排序。未匹配时返回裸 error。

**改进措施**：

1. **模板库扩展**：补充高频军审问句模板，目标扩至 15+ 模板。

2. **歧义消解**（映射"提示词消歧"）：多模板命中时按置信度排序返回候选列表：

```python
def _match_templates(self, query: str) -> List[Dict]:
    candidates = []
    for name, tpl in self.QUERY_TEMPLATES.items():
        for pattern in tpl["patterns"]:
            m = re.search(pattern, query)
            if m:
                candidates.append({
                    "template": name,
                    "confidence": len(m.group(0)) / len(query),
                    "params": m.groups(),
                })
    return sorted(candidates, key=lambda x: -x["confidence"])
```

3. **缺失上下文提示**（映射"上下文澄清"）：未匹配时返回"可提问示例"列表：

```python
if not candidates:
    return {
        "matched": False,
        "message": "未理解您的问题，请参考以下示例提问：",
        "examples": [tpl["description"] for tpl in self.QUERY_TEMPLATES.values()][:5],
    }
```

**舍弃项**：LLM 式自然语言理解、多轮上下文记忆 —— 离线无 LLM，标注未来演进。

**验收**：未匹配时返回示例列表；多模板命中返回排序候选；27 个现有测试全绿。

### 1.3 recommendation 案例库建设

| 属性 | 值 |
|------|----|
| 优先级 | P2 |
| 预计工时 | 20h |
| 风险 | 中（需新增表与迁移） |

**验收**：案例库表迁移成功；推荐结果命中率提升；可按标签检索案例。

### 2.2 业务写操作审计日志补全

| 属性 | 值 |
|------|----|
| 优先级 | **P0**（军队审计必须项） |
| 预计工时 | 16h（v1.0 为 20h，因中间件层已完成，工时下调） |
| 风险 | 中 |
| 涉及文件 | `backend/app/services/work_log_service.py`、`backend/app/middleware/audit_event_handler.py` + 25+ 写操作模块 |

**现状**：
- ✅ **HTTP 请求层审计已完成**：`audit_middleware.py` 已将每次请求落库 `api_access_logs`（含 user_id/username/endpoint/method/status/duration/ip/user_agent）。
- ❌ **业务写操作审计仍缺口**：`write_work_log` 仅被 4 个模块调用（auth/users、projects、school、rural_works，共 12 处），**25+ 个写操作模块**（funds/organization/policy/approval 等）的 POST/PUT/DELETE 无业务级审计日志。

**改进措施**：

1. **抽象 `@audit_write` 装饰器**或 FastAPI 依赖，统一在 POST/PUT/DELETE 入口写审计日志：

```python
def audit_write(target_type: str, action_field: str = "action"):
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, db: Session = Depends(get_db),
                          current_user: User = Depends(get_current_active_user), **kwargs):
            result = await func(*args, db=db, current_user=current_user, **kwargs)
            entity_id = getattr(result, "id", None)
            write_work_log(
                db=db, log_type="audit", action=kwargs.get(action_field, "unknown"),
                entity_id=entity_id, entity_name=target_type,
                operator=current_user.username,
            )
            return result
        return wrapper
    return decorator
```

2. **优先覆盖**：funds、fund_lifecycle、approval、policy、organization（经费/审批/组织为军队审计重点）。

3. 评估是否能通过 `audit_event_handler.py` 中间件统一拦截而非装饰器。

**验收**：所有写操作在 `work_logs` 表留痕；可按操作人/时间/类型检索审计记录。

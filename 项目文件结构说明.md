## 📁 一、根目录 — 项目最外层一眼看懂

```
军队乡村振兴管理系统/
├── backend/          ← 后端（Python）— 系统的"大脑"，管数据存储和业务逻辑
├── frontend/         ← 前端（Vue）— 系统的"脸"，管页面展示和用户操作
├── electron/         ← 桌面壳 — 把网页包成 Windows 桌面程序
├── resources/        ← 静态资源 — 图标、地图、前端编译后的文件
├── database/         ← 数据库参考 — 初始建表SQL（仅供参考）
├── docs/             ← 项目文档
├── scripts/          ← 运维脚本 — 启动、备份、部署
├── build-scripts/    ← 构建脚本 — 打包 exe 和 deb 安装包
├── build/            ← 构建配置 — NSIS 安装包制作
├── deploy/           ← 部署配置 — Nginx、supervisor等
├── docker/           ← Docker 容器化部署
├── k8s/              ← Kubernetes 云原生部署
├── data/             ← 运行时数据 — SQLite数据库文件
├── data_sync/        ← 数据同步目录 — 多机协同交换数据
├── logs/             ← 日志文件
├── uploads/          ← 用户上传的文件
├── exports/          ← 用户导出的文件
├── tools/            ← 辅助小工具
├── installers/       ← 打包好的安装程序放这里
├── README.md         ← 项目总说明（新人先看这个）
├── CLAUDE.md         ← AI助手的说明书（告诉AI怎么帮你写代码）
├── CHANGELOG.md      ← 版本更新记录
├── 启动系统.bat       ← Windows一键启动
├── Makefile          ← Linux/Mac 构建命令
├── Dockerfile        ← 构建Docker镜像的配方
├── docker-compose.yml← 一键启动多个Docker容器
├── .env.example      ← 配置模板（真实的.env不提交git）
└── .gitignore        ← 告诉Git哪些文件不要管
```

## 📁 二、Backend（后端）— 系统的"大脑"

```
backend/
├── app/                     ← 核心代码（重点看这里）
│   ├── api/v1/              ← API 接口层 —"服务员"，接收前端请求
│   │   └── auth/            ←   登录/注册相关接口
│   ├── core/                ← 核心层 —"基础设施"，配置、安全、数据库
│   ├── models/              ← 模型层 —"数据表"，一张表一个文件
│   ├── schemas/             ← 校验层 — 检查前端发来的数据格式对不对
│   ├── services/            ← 业务逻辑层 —"真正干活的地方"
│   │   ├── ai/              ←   AI 相关服务
│   │   ├── domain/          ←   领域驱动设计层
│   │   ├── funding/         ←   经费子服务
│   │   ├── project/         ←   项目子服务
│   │   ├── sentiment/       ←   舆情分析子服务
│   │   ├── village/         ←   帮扶村子服务
│   │   └── zero_trust/      ←   零信任安全子服务
│   ├── middleware/           ← 中间件 —"安检通道"，每个请求都要经过
│   └── utils/               ← 工具函数 —"工具箱"
├── tests/                   ← 测试代码（保证代码质量）
│   ├── unit/                ←   单元测试（测单个函数/类）
│   ├── integration/         ←   集成测试（测多个模块协作）
│   └── security/            ←   安全测试
├── migrations/              ← 数据库迁移记录（Alembic 管理）
├── start.py                 ← 启动入口（python start.py）
├── main.py                  ← FastAPI 应用主文件
└── requirements.txt         ← Python 依赖清单
```

### 🔒 `app/middleware/` — 中间件（每个请求的"安检通道"）

| `auth.py` | **认证** — 你是谁？登录了没？ |
| `rbac.py` | **授权** — 你有什么权限？ |

## 📁 五、数据怎么流动的？（小白必读）

```
用户在浏览器里操作
    │
    ▼
前端 views/ 页面    ← 你看到的界面
    │  调用 API
    ▼
前端 api/ 请求层    ← 发送 HTTP 请求
    │  HTTP 请求（JSON）
    ▼
后端 api/v1/ 路由   ← 接收请求，检查权限
    │  调用 service
    ▼
后端 services/ 业务  ← 处理业务逻辑
    │  读写数据
    ▼
后端 models/ 数据库  ← SQLite 数据库文件
```

**举个实际例子**：用户在页面上点"添加帮扶村"

1. `views/supported-villages/` 页面收集表单数据
2. `api/supportedVillage.ts` 发送 POST 请求到 `/api/v1/supported-villages`
3. `api/v1/supported_village.py` 接收请求，验证数据
4. `services/supported_village_service.py` 处理业务逻辑
5. `models/supported_village.py` 写入 `supported_villages` 表
6. 返回成功消息给前端，页面弹出"创建成功"

# 军队乡村振兴管理系统 — 项目文件结构说明

> 面向新手：本文档用通俗语言解释项目中每个文件和文件夹的作用。
> 即使你不懂编程，也能通过本文档了解系统的"骨架"。

## 📂 项目根目录（你打开项目后最先看到的地方）

| 文件/文件夹 | 作用（通俗解释） |
|-------------|------------------|
| `backend/` | **后端代码** — 系统的"大脑"，Python 编写，处理数据存储、计算、API 接口 |
| `frontend/` | **前端代码** — 系统的"脸面"，Vue 3 编写，你在浏览器里看到的界面 |
| `electron/` | **桌面壳** — 把网页界面包装成 Windows/Linux 桌面程序 |

## 🧠 backend/ — 后端代码

> 系统的大脑：Python FastAPI 编写，负责处理数据、提供 API 接口。

### 后端根目录文件

| 文件 | 作用 |
|------|------|
| `start.py` | **启动入口** — 运行这个文件启动后端服务（集成 WinError 10054 修复） |
| `start_safe.py` | **安全启动** — 带额外检查和自动恢复的启动方式 |
| `init_db.py` | **数据库初始化** — 创建数据库表、填充初始数据 |
| `requirements.txt` | **依赖列表** — 后端需要安装的 Python 包（主环境） |
| `requirements-dev.txt` | **开发依赖** — 开发调试时才需要的额外包 |
| `requirements-docker.txt` | **Docker 依赖** — Docker 构建环境专用依赖 |
| `requirements-minimal.txt` | **最小依赖** — 精简运行所需的最小包集合 |
| `requirements-prod.txt` | **生产依赖** — 生产环境部署专用依赖 |
| `alembic.ini` | **数据库迁移配置** — Alembic 管理数据库表结构的版本升级 |
| `pytest.ini` | **测试配置** — pytest 运行参数和插件配置 |

## 🎨 frontend/ — 前端代码

> 系统的脸面：Vue 3 + TypeScript 编写，你在浏览器里看到的所有界面都在这里。

### 前端根目录文件

| 文件 | 作用 |
|------|------|
| `index.html` | **入口页面** — 浏览器最先加载的 HTML 文件 |
| `package.json` | **项目配置** — 前端项目名称、版本、依赖包列表 |
| `vite.config.ts` | **构建配置** — 定义如何编译、打包、压缩前端代码 |

| `playwright.config.ts` | **E2E 测试配置** — 模拟用户操作浏览器的自动化测试 |
| `tsconfig.json` | **TypeScript 配置** — 类型检查的严格程度 |

## 🚀 新手快速上手

### 想让系统跑起来

```bash
# 方法1：双击 Windows 批处理文件
双击 "启动系统.bat"

# 方法2：命令行启动
cd backend && python start.py    # 启动后端
cd frontend && npm run dev       # 启动前端
```

然后浏览器打开 **http://localhost:5173**，用 `admin` / `admin123` 登录。

### 想了解某个功能

1. 先看 `docs/` 里的用户手册
2. 再看 `frontend/src/views/` 找到对应页面代码
3. 然后看 `backend/app/api/v1/` 找到对应接口代码
4. 最后看 `backend/app/services/` 找到业务逻辑

### 想修改界面

1. 找到 `frontend/src/views/` 对应的 `.vue` 文件
2. 修改 `<template>` 部分的 HTML（界面布局）
3. 修改 `<script>` 部分的逻辑（交互行为）
4. 修改 `<style>` 部分的样式（颜色、大小等）
5. 保存后浏览器自动刷新（Vite 热更新）

### 想修改接口

1. 找到 `backend/app/api/v1/` 对应的 `.py` 文件
2. 修改路由函数
3. 重启后端服务
4. 用浏览器或 curl 测试新接口
